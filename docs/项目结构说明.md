# 项目结构说明

## 项目定位
当前仓库为跨境物流作业系统后端单仓库，采用单体架构与模块化组织方式。代码按 `controller / service / model / dto / route` 分层，配套测试工具和项目文档均已落在仓库内。

---

## 目录结构

### `/controllers`
控制器层，负责：
- 接收 HTTP 请求
- 绑定参数
- 调用服务层
- 返回统一响应

当前主要文件：
- `auth_controller.go`：认证与个人信息
- `user_controller.go`：用户管理
- `role_controller.go`：角色与权限
- `order_controller.go`：订单管理
- `station_controller.go`：站点与库存
- `warehouse_controller.go`：入库/出库
- `sorting_controller.go`：分拣规则、任务、扫描、统计
- `transport_controller.go`：车辆、运输任务、装卸、监控、成本
- `tracking_controller.go`：追踪记录、时间轴、预警
- `exception_controller.go`：异常创建、处理、关闭、统计
- `dispatch_controller.go`：路径优化、批次调度、运输计划、调度建议

### `/services`
业务逻辑层，负责：
- 业务规则校验
- 数据读写与事务控制
- 状态流转与统计分析

当前主要文件：
- `user_service.go`
- `order_service.go`
- `order_state_machine.go`
- `station_service.go`
- `warehouse_service.go`
- `sorting_service.go`
- `transport_service.go`
- `tracking_service.go`
- `exception_service.go`
- `dispatch_service.go`

### `/models`
数据模型层，对应数据库表结构。

当前主要模型：
- 用户：`user.go`
- 订单：`order.go`、`order_status_log.go`
- 站点与库存：`station.go`、`warehouse.go`、`inventory_check.go`
- 分拣：`sorting.go`
- 运输与调度：`transport.go`、`delivery.go`
- 追踪与异常：`tracking.go`、`exception.go`

### `/dto`
请求/响应对象定义，供 controller 和 service 之间传递结构化数据。

当前主要 DTO：
- `user.go`
- `order.go`
- `station.go`
- `warehouse.go`
- `sorting.go`
- `transport.go`
- `tracking.go`
- `exception.go`
- `common.go`

### `/routes`
路由注册与分组。

当前主要路由组：
- `/api/auth`
- `/api/users`
- `/api/orders`
- `/api/stations`
- `/api/warehouse`
- `/api/sorting`
- `/api/transport`
- `/api/tracking`
- `/api/exceptions`
- `/api/dispatch`
- `/api/admin`

### `/middleware`
通用中间件：
- `auth.go`：JWT 与权限控制
- `cors.go`：跨域处理
- `logger.go`：日志记录
- `recovery.go`：异常恢复

### `/database`
数据库与缓存初始化：
- `database.go`：MySQL 初始化、连接池、自动迁移
- `redis.go`：Redis 初始化与关闭

### `/utils`
工具函数：
- `jwt.go`：JWT 生成与解析
- `password.go`：密码哈希与校验
- `response.go`：统一响应封装
- `time.go`：时间格式化工具

### `/testutil`
自动化测试辅助工具：
- 自动加载 `.env`
- 初始化数据库连接
- 初始化 JWT 密钥
- 创建测试用户、订单、站点、车辆
- 在测试后清理测试数据

当前主要文件：
- `testenv.go`

### `/frontend`
前端项目骨架，当前已接入：
- Vue 3
- Vite
- Element Plus
- Vue Router
- Pinia
- Axios

当前前端已完成：
- 登录页
- 主框架布局
- 路由守卫
- 鉴权状态管理
- 请求拦截与错误处理
### `/docs`
项目文档目录，当前包含：
- 开发计划
- 项目结构说明
- 统一 API 文档
- 自动化测试说明文档
- 联调与回归测试报告
- 各模块功能说明 / 测试说明 / 测试报告

---

## 代码分层关系

```text
HTTP Request
  -> Middleware
  -> Controller
  -> Service
  -> Model / Database
  -> Response
```

### 各层职责
- Controller：处理协议层和参数
- Service：处理业务规则和事务
- Model：描述数据结构
- DTO：描述请求/响应结构
- Middleware：处理认证、权限、日志、异常恢复
