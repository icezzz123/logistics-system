- **写作配合方式**：正文先概述业务链路，再结合图说明各节点的业务含义，图后可自然过渡到核心业务场景分析

#### 3.1.2 核心业务场景分析（约800字）
**写作要点**:
- 订单受理与录入场景

- 分拣作业与路由匹配场景
- 运输调度与在途管理场景

**配图安排**:

- **写作配合方式**：正文按场景顺序展开，图用于强化“多角色协同”的分析重点，并为 3.2 用户角色分析做铺垫

## 配图注意事项

1. **统一风格**: 所有图表使用统一的配色方案和字体
2. **清晰易读**: 字体大小适中（建议10-12号），线条清晰，避免过于复杂
3. **中文标注**: 所有图表使用中文标注
4. **图表编号**: 按顺序编号（图3-1、图3-2、表3-1、表3-2...）
5. **图表标题**: 每个图表下方添加标题和简要说明
6. **引用说明**: 在正文中引用图表（如"如图3-1所示"、"详见表3-1"）
7. **图表位置**: 图表应紧跟相关文字说明之后
8. **分辨率**: 确保图片清晰，建议300dpi以上

## 写作注意事项

### 风格要求
1. **口语化学术风格**: 使用"这块"、"比较"、"挺"等自然表达
2. **避免重复过渡词**: 不要连续使用"在...方面"
3. **长短句结合**: 有详细阐述也有简洁总结
4. **真实内容**: 基于实际实现的系统功能
5. **不虚构内容**: 所有描述必须真实

### 结构要求
1. 每个小节开头简要说明该部分的目的
2. 使用列表、表格等方式提高可读性
3. 适当使用图表辅助说明（如业务流程图、角色协同图）
4. 各功能模块需求描述要具体、可实现
