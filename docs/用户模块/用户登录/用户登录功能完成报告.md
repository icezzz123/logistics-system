## 实现功能

### 1. 用户登录接口

**接口**: `POST /api/auth/login`

**功能**:
- 用户名密码验证
- 密码bcrypt加密验证
- 用户状态检查
- JWT令牌生成（24小时有效期）
- 返回用户基本信息

**实现文件**:
- `controllers/auth_controller.go` - Login()方法
- `services/user_service.go` - Login()方法

### 2. 获取用户信息接口

**接口**: `GET /api/profile`

**功能**:
- 需要JWT认证
- 返回当前登录用户的完整信息
- 包含创建时间和修改时间

**实现文件**:
- `controllers/auth_controller.go` - GetProfile()方法
- `services/user_service.go` - GetUserByID()方法

### 3. JWT认证中间件

**功能**:
- 验证Authorization请求头
- 解析和验证JWT令牌
- 提取用户信息到上下文
- 拦截未授权请求

**实现文件**:
- `middleware/auth.go` - AuthMiddleware()

### 4. JWT工具类

**功能**:
- 生成JWT令牌
- 解析JWT令牌
- 验证令牌有效性

**实现文件**:
- `utils/jwt.go` - GenerateToken(), ParseToken()

### 5. 密码验证工具

**功能**:
- bcrypt密码验证

**实现文件**:
- `utils/password.go` - CheckPassword()

## 技术实现

### 架构设计

采用三层架构：

```
Controller层 (控制器)
    ↓
Service层 (业务逻辑)
    ↓
Model层 (数据模型)
```

### 核心技术

1. **Gin框架**: Web框架和路由
2. **GORM**: ORM数据库操作
3. **JWT**: 令牌生成和验证
4. **bcrypt**: 密码加密和验证
5. **中间件**: 请求拦截和认证

### 安全特性

1. **密码加密**: 使用bcrypt算法，不存储明文密码
2. **JWT签名**: 使用HS256算法签名，防止篡改
3. **防用户枚举**: 用户名和密码错误返回相同信息
4. **状态检查**: 禁用账号无法登录
5. **令牌过期**: 24小时自动失效
6. **密码不返回**: 响应中不包含密码字段

## API接口

### 1. 登录接口

```
POST /api/auth/login
Content-Type: application/json

Request:
{
  "username": "minuser",
  "password": "123456"
}

Response (200):
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 3,
      "username": "minuser",
      "email": "",
      "phone": "",
      "real_name": "",
      "role": 1,
      "status": 1
    }
  }
}
```

### 2. 获取用户信息接口

```
GET /api/profile
Authorization: Bearer {token}

Response (200):
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 3,
    "username": "minuser",
    "email": "",
    "phone": "",
    "real_name": "",
    "role": 1,
    "status": 1,
    "ctime": 1772273620,
    "mtime": 1772273620
  }
}
```

## 使用示例

### 1. 登录获取Token

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"minuser","password":"123456"}'
```

### 2. 使用Token访问接口

```bash
curl -X GET http://localhost:8080/api/profile \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```
