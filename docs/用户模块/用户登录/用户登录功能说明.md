# 用户登录功能说明

## 功能概述

用户登录功能允许已注册用户通过用户名和密码进行身份验证，成功后获取JWT令牌用于后续API访问。

## API接口

### 1. 用户登录

**接口地址**: `POST /api/auth/login`

**请求参数**:

```json
{
  "username": "minuser",      // 必填，用户名
  "password": "123456"        // 必填，密码
}
```

**成功响应** (200):

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 3,
      "username": "minuser",
      "email": "",
      "phone": "",
      "real_name": "",
      "role": 1,
      "status": 1
    }
  }
}
```

**错误响应**:

- 用户名或密码错误 (401):
```json
{
  "code": 401,
  "message": "用户名或密码错误"
}
```

- 账号已被禁用 (401):
```json
{
  "code": 401,
  "message": "账号已被禁用，请联系管理员"
}
```

- 参数错误 (400):
```json
{
  "code": 400,
  "message": "参数错误: ..."
}
```

### 2. 获取用户信息

**接口地址**: `GET /api/profile`

**请求头**:
```
Authorization: Bearer {token}
```

**成功响应** (200):

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 3,
    "username": "minuser",
    "email": "",
    "phone": "",
    "real_name": "",
    "role": 1,
    "status": 1,
    "ctime": 1772273620,
    "mtime": 1772273620
  }
}
```

**错误响应**:

- 未提供令牌 (401):
```json
{
  "code": 401,
  "message": "未提供认证令牌"
}
```

- 无效令牌 (401):
```json
{
  "code": 401,
  "message": "无效的认证令牌"
}
```

## 业务逻辑

### 登录流程

1. **参数验证**
   - 检查用户名和密码是否为空
   - 去除首尾空格

2. **用户查询**
   - 根据用户名查询数据库
   - 用户不存在返回"用户名或密码错误"

3. **密码验证**
   - 使用bcrypt验证密码
   - 密码错误返回"用户名或密码错误"

4. **状态检查**
   - 检查用户状态是否为启用(status=1)
   - 已禁用返回错误提示

5. **生成令牌**
   - 生成JWT令牌，有效期24小时
   - 令牌包含：用户ID、用户名、角色

6. **返回结果**
   - 返回令牌和用户基本信息
   - 不返回密码字段

### JWT令牌说明

- **算法**: HS256
- **有效期**: 24小时
- **包含信息**:
  - user_id: 用户ID
  - username: 用户名
  - role: 用户角色
  - iss: 签发者 (logistics-system)
  - exp: 过期时间
  - iat: 签发时间

### 认证中间件

所有需要认证的接口都使用`AuthMiddleware`中间件：

1. 从请求头获取`Authorization`字段
2. 验证格式为`Bearer {token}`
3. 解析JWT令牌
4. 验证令牌有效性
5. 将用户信息存入上下文

## 安全特性

1. **密码加密**: 使用bcrypt算法加密存储
2. **令牌签名**: JWT使用密钥签名，防止篡改
3. **错误信息**: 用户名和密码错误返回相同信息，防止用户枚举
4. **状态检查**: 禁用账号无法登录
5. **令牌过期**: 24小时后自动失效

## 使用示例

### 登录获取令牌

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"minuser","password":"123456"}'
```

### 使用令牌访问接口

```bash
curl -X GET http://localhost:8080/api/profile \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```
