# 用户登录代码执行流程

## 完整执行流程

### 1. 客户端发起登录请求

```
POST /api/auth/login
Content-Type: application/json

{
  "username": "minuser",
  "password": "123456"
}
```

### 2. 路由层 (routes/routes.go)

```go
// 公开路由
public := r.Group("/api")
{
    auth := public.Group("/auth")
    {
        auth.POST("/login", authCtrl.Login)  // 匹配到登录路由
    }
}
```

**执行步骤**:
- Gin框架接收HTTP请求
- 匹配路由规则 `/api/auth/login`
- 调用 `authCtrl.Login` 方法

### 3. 控制器层 (controllers/auth_controller.go)

#### 3.1 Login方法入口

```go
func (ctrl *AuthController) Login(c *gin.Context) {
    var req LoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        utils.BadRequest(c, "参数错误: "+err.Error())
        return
    }
    // ...
}
```

**执行步骤**:
1. 声明 `LoginRequest` 结构体变量
2. 使用 `ShouldBindJSON` 绑定JSON数据
3. 验证必填字段 (username, password)
4. 如果验证失败，返回400错误

#### 3.2 转换为DTO并调用服务层

```go
// 转换为DTO
loginDTO := &dto.LoginRequest{
    Username: req.Username,
    Password: req.Password,
}

// 调用服务层进行登录
userService := services.NewUserService()
user, token, err := userService.Login(loginDTO)
if err != nil {
    utils.Error(c, 401, err.Error())
    return
}
```

**执行步骤**:
1. 创建 `dto.LoginRequest` 对象
2. 实例化 `UserService`
3. 调用 `userService.Login()` 方法
4. 如果失败，返回401错误

#### 3.3 返回成功响应

```go
utils.Success(c, gin.H{
    "token": token,
    "user": gin.H{
        "id":        user.ID,
        "username":  user.Username,
        "email":     user.Email,
        "phone":     user.Phone,
        "real_name": user.RealName,
        "role":      user.Role,
        "status":    user.Status,
    },
})
```

**执行步骤**:
1. 构造响应数据
2. 调用 `utils.Success` 返回200响应
3. 包含token和用户信息

### 4. 服务层 (services/user_service.go)

#### 4.1 Login方法 - 参数验证

```go
func (s *UserService) Login(req *dto.LoginRequest) (*models.User, string, error) {
    // 1. 参数验证
    if strings.TrimSpace(req.Username) == "" {
        return nil, "", errors.New("用户名不能为空")
    }
    if strings.TrimSpace(req.Password) == "" {
        return nil, "", errors.New("密码不能为空")
    }
    // ...
}
```

**执行步骤**:
1. 去除用户名首尾空格并检查是否为空
2. 去除密码首尾空格并检查是否为空
3. 如果为空，返回错误

#### 4.2 查找用户

```go
// 2. 查找用户
var user models.User
if err := database.DB.Where("username = ?", req.Username).First(&user).Error; err != nil {
    if err == gorm.ErrRecordNotFound {
        return nil, "", errors.New("用户名或密码错误")
    }
    return nil, "", errors.New("数据库查询失败")
}
```

**执行步骤**:
1. 声明 `models.User` 变量
2. 使用GORM的 `Where` 和 `First` 查询用户
3. 生成SQL: `SELECT * FROM users WHERE username = 'minuser' LIMIT 1`
4. 如果用户不存在，返回"用户名或密码错误"
5. 如果数据库错误，返回"数据库查询失败"

#### 4.3 验证密码

```go
// 3. 验证密码
if !utils.CheckPassword(req.Password, user.Password) {
    return nil, "", errors.New("用户名或密码错误")
}
```

**执行步骤**:
1. 调用 `utils.CheckPassword` 验证密码
2. 使用bcrypt比对明文密码和哈希密码
3. 如果不匹配，返回"用户名或密码错误"

#### 4.4 检查用户状态

```go
// 4. 检查用户状态
if user.Status != 1 {
    return nil, "", errors.New("账号已被禁用，请联系管理员")
}
```

**执行步骤**:
1. 检查 `user.Status` 字段
2. 如果不等于1（启用），返回禁用错误

#### 4.5 生成JWT令牌

```go
// 5. 生成token
token, err := utils.GenerateToken(user.ID, user.Username, int(user.Role), 24*3600)
if err != nil {
    return nil, "", errors.New("生成令牌失败")
}
```

**执行步骤**:
1. 调用 `utils.GenerateToken` 生成JWT
2. 传入用户ID、用户名、角色、有效期(24小时)
3. 如果失败，返回错误

#### 4.6 清除密码并返回

```go
// 6. 清除密码字段
user.Password = ""

return &user, token, nil
```

**执行步骤**:
1. 将 `user.Password` 设为空字符串
2. 返回用户对象、令牌、nil错误

### 5. JWT工具 (utils/jwt.go)

#### 5.1 GenerateToken方法

```go
func GenerateToken(userID uint, username string, role int, expireSeconds int) (string, error) {
    claims := Claims{
        UserID:   userID,
        Username: username,
        Role:     role,
        RegisteredClaims: jwt.RegisteredClaims{
            Issuer:    "logistics-system",
            ExpiresAt: jwt.NewNumericDate(time.Now().Add(time.Duration(expireSeconds) * time.Second)),
            IssuedAt:  jwt.NewNumericDate(time.Now()),
        },
    }
    
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString(jwtSecret)
}
```

**执行步骤**:
1. 创建 `Claims` 结构体，包含：
   - UserID: 用户ID
   - Username: 用户名
   - Role: 角色
   - Issuer: 签发者 "logistics-system"
   - ExpiresAt: 过期时间（当前时间+24小时）
   - IssuedAt: 签发时间（当前时间）
2. 使用HS256算法创建JWT
3. 使用密钥签名并返回token字符串

### 6. 密码验证工具 (utils/password.go)

#### 6.1 CheckPassword方法

```go
func CheckPassword(password, hash string) bool {
    err := bcrypt.CompareHashAndPassword([]byte(hash), []byte(password))
    return err == nil
}
```

**执行步骤**:
1. 调用bcrypt的 `CompareHashAndPassword`
2. 比对哈希密码和明文密码
3. 如果匹配返回true，否则返回false

### 7. 响应工具 (utils/response.go)

#### 7.1 Success方法

```go
func Success(c *gin.Context, data interface{}) {
    c.JSON(http.StatusOK, Response{
        Code:    200,
        Message: "success",
        Data:    data,
    })
}
```

**执行步骤**:
1. 构造 `Response` 结构体
2. 设置code=200, message="success"
3. 使用 `c.JSON` 返回HTTP 200响应

## 使用Token访问受保护接口流程

### 1. 客户端发起请求

```
GET /api/profile
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 2. 认证中间件 (middleware/auth.go)

#### 2.1 AuthMiddleware入口

```go
func AuthMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        authHeader := c.GetHeader("Authorization")
        if authHeader == "" {
            utils.Error(c, 401, "未提供认证令牌")
            c.Abort()
            return
        }
        // ...
    }
}
```

**执行步骤**:
1. 从请求头获取 `Authorization` 字段
2. 如果为空，返回401错误并中止请求

#### 2.2 解析Token格式

```go
parts := strings.SplitN(authHeader, " ", 2)
if len(parts) != 2 || parts[0] != "Bearer" {
    utils.Error(c, 401, "认证令牌格式错误")
    c.Abort()
    return
}
```

**执行步骤**:
1. 按空格分割Authorization字段
2. 验证格式为 "Bearer {token}"
3. 如果格式错误，返回401并中止

#### 2.3 验证Token

```go
claims, err := utils.ParseToken(parts[1])
if err != nil {
    utils.Error(c, 401, "无效的认证令牌")
    c.Abort()
    return
}
```

**执行步骤**:
1. 调用 `utils.ParseToken` 解析JWT
2. 验证签名和过期时间
3. 如果无效，返回401并中止

#### 2.4 存储用户信息到上下文

```go
c.Set("user_id", claims.UserID)
c.Set("username", claims.Username)
c.Set("role", claims.Role)
c.Next()
```

**执行步骤**:
1. 将用户ID、用户名、角色存入Gin上下文
2. 调用 `c.Next()` 继续执行后续处理器

### 3. 控制器获取用户信息

```go
func (ctrl *AuthController) GetProfile(c *gin.Context) {
    userID, exists := c.Get("user_id")
    if !exists {
        utils.Error(c, 401, "未授权访问")
        return
    }
    
    userService := services.NewUserService()
    user, err := userService.GetUserByID(userID.(uint))
    // ...
}
```

**执行步骤**:
1. 从上下文获取 `user_id`
2. 调用服务层查询用户详细信息
3. 返回用户数据

## 数据流转图

```
客户端请求
    ↓
Gin路由匹配 (routes/routes.go)
    ↓
控制器层 (controllers/auth_controller.go)
    ├─ 参数绑定和验证
    └─ 调用服务层
        ↓
服务层 (services/user_service.go)
    ├─ 参数验证
    ├─ 数据库查询用户 (database/database.go + GORM)
    ├─ 密码验证 (utils/password.go + bcrypt)
    ├─ 状态检查
    ├─ 生成JWT (utils/jwt.go)
    └─ 返回结果
        ↓
控制器层构造响应
    ↓
响应工具返回JSON (utils/response.go)
    ↓
客户端收到响应
```

## 关键技术点

1. **Gin框架**: 路由、参数绑定、中间件
2. **GORM**: 数据库查询
3. **bcrypt**: 密码加密和验证
4. **JWT**: 令牌生成和解析
5. **中间件**: 认证拦截
6. **分层架构**: Controller → Service → Model
