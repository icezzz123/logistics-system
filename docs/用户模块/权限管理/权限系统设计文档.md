# 权限系统设计文档

## 概述

本系统采用基于角色的访问控制（RBAC）模型，通过角色-权限映射实现细粒度的权限管理。

## 设计原则

1. **最小权限原则**：用户只拥有完成工作所需的最小权限集
2. **角色分离**：不同角色拥有不同的权限范围
3. **可扩展性**：权限系统易于扩展新的权限和角色
4. **安全性**：所有敏感操作都需要权限验证

## 角色定义

### 1. 客户（Customer）- Role ID: 1

**职责**：下单、查询订单、跟踪物流

**权限**：
- `order:view` - 查看订单
- `order:create` - 创建订单
- `tracking:view` - 查看物流跟踪

**使用场景**：
- 普通用户注册后的默认角色
- 可以下单并跟踪自己的订单
- 无法访问系统管理功能

### 2. 快递员（Courier）- Role ID: 2

- `delivery:update` - 更新配送状态
- `tracking:view` - 查看物流跟踪
- `tracking:update` - 更新物流跟踪

**使用场景**：

## 权限检查流程

```
用户请求
    ↓
JWT认证中间件（验证Token）
    ↓
提取用户角色
    ↓
权限中间件（检查权限）
    ↓
查询角色权限映射
    ↓
判断是否拥有所需权限
    ↓
允许/拒绝访问
```

## 实现细节

### 配置文件

**位置**：`config/permissions.go`

**核心函数**：
- `HasPermission(role, permission)` - 检查角色是否拥有权限
- `GetRolePermissions(role)` - 获取角色的所有权限
- `GetRoleName(role)` - 获取角色名称
- `IsValidRole(role)` - 验证角色ID是否有效

### 中间件

**位置**：`middleware/auth.go`

**中间件函数**：
- `RequirePermission(perm)` - 要求特定权限
- `RequireAnyPermission(perms...)` - 要求任一权限
- `RequireAllPermissions(perms...)` - 要求所有权限

**辅助函数**：
- `HasPermission(c, perm)` - 检查当前用户权限
- `GetUserPermissions(c)` - 获取当前用户所有权限

## 使用示例

### 1. 在路由中使用

```go
// 要求订单查看权限
orders.GET("", middleware.RequirePermission(config.PermOrderView), orderCtrl.GetOrderList)

// 要求订单创建权限
orders.POST("", middleware.RequirePermission(config.PermOrderCreate), orderCtrl.CreateOrder)

// 要求管理员权限
admin.Use(middleware.RequireAdmin())
```

### 2. 在控制器中检查

```go
func (ctrl *OrderController) UpdateOrder(c *gin.Context) {
    // 检查是否有更新权限
    if !middleware.HasPermission(c, config.PermOrderUpdate) {
        utils.Forbidden(c, "无权限更新订单")
        return
    }
    
    // 业务逻辑...
}
```

### 3. 查询用户权限

```go
// 获取当前用户的所有权限
permissions := middleware.GetUserPermissions(c)

// 返回给前端
utils.Success(c, gin.H{
    "permissions": permissions,
})
```

## 扩展指南

### 添加新权限

1. 在`config/permissions.go`中定义权限常量：
```go
const (
    PermNewFeatureView Permission = "newfeature:view"
)
```

2. 将权限添加到相应角色的权限列表：
```go
var RolePermissions = map[int][]Permission{
    5: { // 站点管理员
        PermNewFeatureView,
        // ... 其他权限
    },
}
```

### 添加新角色

1. 在`models/user.go`中定义角色常量：
```go
const (
    RoleNewRole UserRole = 8
)
```

2. 在`config/permissions.go`中添加角色权限映射：
```go
var RolePermissions = map[int][]Permission{
    8: { // 新角色
        PermOrderView,
        // ... 权限列表
    },
}

var RoleNames = map[int]string{
    8: "新角色名称",
}
```

## 安全建议

1. **最小权限原则**：只授予必要的权限
2. **定期审计**：定期检查用户权限是否合理
3. **权限分离**：敏感操作需要多个权限
4. **日志记录**：记录所有权限检查和敏感操作
5. **动态权限**：考虑实现基于资源所有权的动态权限

## 性能优化

1. **权限缓存**：将用户权限缓存到Redis
2. **批量检查**：一次检查多个权限
3. **权限预加载**：登录时加载用户权限到Token
4. **索引优化**：数据库角色字段添加索引

## 测试建议

1. **单元测试**：测试每个权限检查函数
2. **集成测试**：测试完整的权限验证流程
3. **边界测试**：测试无权限、部分权限、全权限场景
4. **性能测试**：测试高并发下的权限检查性能
