### 3. 权限中间件

**文件**：`middleware/auth.go`

**中间件函数**：
- `RequirePermission(perm)` - 要求特定权限
- `RequireAnyPermission(perms...)` - 要求任一权限
- `RequireAllPermissions(perms...)` - 要求所有权限
- `RequireAdmin()` - 要求管理员权限
- `RequireStaff()` - 要求员工权限
- `RequireManager()` - 要求管理层权限

**辅助函数**：
- `HasPermission(c, perm)` - 检查当前用户权限
- `GetUserPermissions(c)` - 获取当前用户所有权限
- `CheckPermission(c, roles...)` - 检查用户角色
- `IsAdmin(c)` - 检查是否为管理员
- `IsStaff(c)` - 检查是否为员工
- `IsManager(c)` - 检查是否为管理层
- `GetCurrentUserID(c)` - 获取当前用户ID
- `GetCurrentUsername(c)` - 获取当前用户名
- `GetCurrentUserRole(c)` - 获取当前用户角色

## 技术实现

### 架构设计

```
权限配置层 (config/permissions.go)
    ↓
中间件层 (middleware/auth.go)
    ↓
路由层 (routes/routes.go)
    ↓
控制器层 (controllers/*.go)
```

### 核心技术

1. **基于角色的访问控制（RBAC）**
   - 角色-权限映射
   - 静态权限配置
   - 内存查询，性能高效

2. **中间件模式**
   - Gin框架中间件
   - 可组合的权限检查
   - 支持多种权限验证策略

3. **权限常量**
   - 类型安全的权限定义
   - 避免字符串硬编码
   - 易于维护和扩展

### 安全特性

1. **JWT认证**：所有权限检查前先验证JWT
2. **角色验证**：基于用户角色进行权限判断
3. **细粒度控制**：支持资源级权限控制
4. **权限组合**：支持"任一"和"全部"权限检查
5. **拒绝优先**：无权限时立即拒绝访问

## 使用示例

### 1. 在路由中使用权限中间件

```go
// 要求订单查看权限
orders.GET("", 
    middleware.RequirePermission(config.PermOrderView),
    orderCtrl.GetOrderList)

// 要求管理员权限
admin.Use(middleware.RequireAdmin())
```

### 2. 在控制器中检查权限

```go
func (ctrl *OrderController) UpdateOrder(c *gin.Context) {
    // 检查是否有更新权限
    if !middleware.HasPermission(c, config.PermOrderUpdate) {
        utils.Forbidden(c, "无权限更新订单")
        return
    }
    
    // 业务逻辑...
}
```

### 3. 前端查询权限

```javascript
// 获取用户权限
const response = await fetch('/api/permissions', {
    headers: {
        'Authorization': `Bearer ${token}`
    }
});
const data = await response.json();
const permissions = data.data.permissions;

// 根据权限显示UI
if (permissions.includes('order:create')) {
    showCreateButton();
}
```

## 扩展性

### 添加新权限

1. 在`config/permissions.go`中定义权限常量
2. 将权限添加到相应角色的权限列表
3. 在路由或控制器中使用新权限

### 添加新角色

1. 在`models/user.go`中定义角色常量
2. 在`config/permissions.go`中添加角色权限映射
3. 在`RoleNames`中添加角色名称

### 实现动态权限

当前系统使用静态权限配置，如需动态权限：
1. 创建权限表和角色权限关联表
2. 修改`HasPermission`函数查询数据库
3. 添加权限管理API接口
4. 实现权限缓存机制

## 性能优化建议

1. **权限缓存**：将用户权限缓存到Redis
2. **Token包含权限**：在JWT中包含权限列表
3. **权限位图**：使用位图存储权限，提高检查效率
4. **批量检查**：一次检查多个权限

## 安全建议

1. **最小权限原则**：只授予必要的权限
2. **定期审计**：定期检查用户权限是否合理
3. **权限分离**：敏感操作需要多个权限
4. **日志记录**：记录所有权限检查和拒绝事件
5. **资源所有权**：除了角色权限，还要检查资源归属
