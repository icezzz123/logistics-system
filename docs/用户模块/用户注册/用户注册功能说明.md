# 用户注册功能说明

## 功能概述

用户注册功能允许新用户创建账号，支持用户名、密码、邮箱、手机号和真实姓名的注册。

## API接口

### 注册接口

**请求地址**: `POST /api/auth/register`

**请求头**:
```
Content-Type: application/json
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 | 验证规则 |
|--------|------|------|------|----------|
| username | string | 是 | 用户名 | 3-50个字符，只能包含字母、数字、下划线 |
| password | string | 是 | 密码 | 6-50个字符 |
| email | string | 否 | 邮箱 | 标准邮箱格式 |
| phone | string | 否 | 手机号 | 11位中国大陆手机号（1开头） |
| real_name | string | 否 | 真实姓名 | 最多50个字符 |

**请求示例**:
```json
{
  "username": "zhangsan",
  "password": "123456",
  "email": "zhangsan@example.com",
  "phone": "13800138000",
  "real_name": "张三"
}
```

**成功响应** (200):
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "zhangsan",
    "email": "zhangsan@example.com",
    "phone": "13800138000",
    "real_name": "张三",
    "role": 1
  }
}
```

**失败响应** (400):
```json
{
  "code": 400,
  "message": "用户名已存在"
}
```

## 业务逻辑

### 1. 参数验证

#### 用户名验证
- 长度：3-50个字符
- 格式：只能包含字母、数字、下划线
- 唯一性：不能与已有用户名重复

#### 密码验证
- 长度：6-50个字符
- 存储：使用bcrypt加密存储

#### 邮箱验证（可选）
- 格式：标准邮箱格式（xxx@xxx.xxx）
- 唯一性：如果提供，不能与已有邮箱重复

#### 手机号验证（可选）
- 格式：11位中国大陆手机号（1开头）
- 唯一性：如果提供，不能与已有手机号重复

#### 真实姓名验证（可选）
- 长度：最多50个字符

### 2. 注册流程

```
1. 接收并验证请求参数
   ↓
2. 检查用户名是否已存在
   ↓
3. 检查邮箱是否已被使用（如果提供）
   ↓
4. 检查手机号是否已被使用（如果提供）
   ↓
5. 使用bcrypt加密密码
   ↓
6. 创建用户记录
   - 默认角色：客户（role=1）
   - 默认状态：启用（status=1）
   ↓
7. 返回用户信息（不包含密码）
```

### 3. 默认值

- **角色（role）**: 1（客户）
- **状态（status）**: 1（启用）
- **创建时间（ctime）**: 自动生成
- **修改时间（mtime）**: 自动生成

## 错误码说明

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 400 | 参数错误 | 请求参数格式不正确 |
| 400 | 用户名长度不能少于3个字符 | 用户名太短 |
| 400 | 用户名长度不能超过50个字符 | 用户名太长 |
| 400 | 用户名只能包含字母、数字和下划线 | 用户名格式不正确 |
| 400 | 密码长度不能少于6个字符 | 密码太短 |
| 400 | 密码长度不能超过50个字符 | 密码太长 |
| 400 | 邮箱格式不正确 | 邮箱格式验证失败 |
| 400 | 手机号格式不正确 | 手机号格式验证失败 |
| 400 | 用户名已存在 | 用户名重复 |
| 400 | 邮箱已被使用 | 邮箱重复 |
| 400 | 手机号已被使用 | 手机号重复 |
| 500 | 密码加密失败 | 服务器内部错误 |
| 500 | 用户创建失败 | 数据库操作失败 |

## 代码结构

### 分层架构

```
Controller层 (controllers/auth_controller.go)
    ↓ 接收HTTP请求，参数绑定
Service层 (services/user_service.go)
    ↓ 业务逻辑处理，数据验证
Model层 (models/user.go)
    ↓ 数据模型定义
Database层 (database/database.go)
    ↓ 数据库操作
```

### 关键文件

1. **controllers/auth_controller.go**
   - `Register()` - 注册控制器方法
   - 负责HTTP请求处理和响应

2. **services/user_service.go**
   - `Register()` - 注册业务逻辑
   - `validateRegisterParams()` - 参数验证
   - 负责核心业务逻辑

3. **dto/user.go**
   - `RegisterRequest` - 注册请求DTO
   - 定义请求参数结构

4. **models/user.go**
   - `User` - 用户模型
   - 定义数据库表结构

5. **utils/password.go**
   - `HashPassword()` - 密码加密
   - `CheckPassword()` - 密码验证

## 安全性

### 密码安全
- 使用bcrypt算法加密密码
- 密码不会以明文形式存储
- 响应中不返回密码字段

### 输入验证
- 所有输入参数都经过严格验证
- 防止SQL注入（使用GORM参数化查询）
- 防止XSS攻击（输入过滤）

### 唯一性检查
- 用户名唯一性检查
- 邮箱唯一性检查（如果提供）
- 手机号唯一性检查（如果提供）
