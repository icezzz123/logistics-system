# 用户注册代码执行流程详解

## 概览

用户注册请求从HTTP请求到数据库存储的完整执行路径：

```
HTTP请求 → 中间件 → 路由 → 控制器 → 服务层 → 数据库 → 响应
```

---

### 第2步：通过中间件处理

**文件**: `middleware/recovery.go`, `middleware/logger.go`, `middleware/cors.go`

#### 2.1 RecoveryMiddleware（异常恢复）
```go
func RecoveryMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        defer func() {
            if err := recover(); err != nil {
                // 捕获panic，记录日志，返回500错误
                log.Printf("Panic recovered: %v", err)
                c.JSON(500, gin.H{"error": "Internal Server Error"})
                c.Abort()
            }
        }()
        c.Next()
    }
}
```

#### 2.2 LoggerMiddleware（日志记录）
```go
func LoggerMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        start := time.Now()
        c.Next()
        // 记录请求信息：方法、路径、状态码、耗时
        log.Printf("[%s] %s - %d (%v)",
            c.Request.Method,
            c.Request.URL.Path,
            c.Writer.Status(),
            time.Since(start))
    }
}
```

#### 2.3 CORSMiddleware（跨域处理）
```go
func CORSMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Writer.Header().Set("Access-Control-Allow-Origin", "*")
        c.Writer.Header().Set("Access-Control-Allow-Methods", "POST, GET, OPTIONS, PUT, DELETE")
        // ... 其他CORS头
        c.Next()
    }
}
```

### 第4步：控制器层处理

**文件**: `controllers/auth_controller.go`

```go
// Register 用户注册
func (ctrl *AuthController) Register(c *gin.Context) {
    // 4.1 定义请求结构体
    var req RegisterRequest
    
    // 4.2 绑定并验证JSON参数
    if err := c.ShouldBindJSON(&req); err != nil {
        utils.BadRequest(c, "参数错误: "+err.Error())
        return
    }
    // Gin会自动验证：
    // - username: required（必填）
    // - password: required, min=6（必填，最少6字符）
    
    // 4.3 转换为DTO对象
    registerDTO := &dto.RegisterRequest{
        Username: req.Username,
        Password: req.Password,
        Email:    req.Email,
        Phone:    req.Phone,
        RealName: req.RealName,
    }
    
    // 4.4 调用服务层进行注册
    userService := services.NewUserService()
    user, err := userService.Register(registerDTO)
    if err != nil {
        utils.BadRequest(c, err.Error())
        return
    }
    
    // 4.5 返回成功响应
    utils.SuccessWithMessage(c, "注册成功", gin.H{
        "id":        user.ID,
        "username":  user.Username,
        "email":     user.Email,
        "phone":     user.Phone,
        "real_name": user.RealName,
        "role":      user.Role,
    })
}
```

**数据流**:
```
HTTP JSON → RegisterRequest → dto.RegisterRequest → Service层
```

### 第5步：服务层业务逻辑

**文件**: `services/user_service.go`

```go
// Register 用户注册
func (s *UserService) Register(req *dto.RegisterRequest) (*models.User, error) {
    
    // 5.1 参数验证
    if err := s.validateRegisterParams(req); err != nil {
        return nil, err
    }
    // 验证内容：
    // - 用户名：3-50字符，只能包含字母数字下划线
    // - 密码：6-50字符
    // - 邮箱：标准邮箱格式（如果提供）
    // - 手机号：11位1开头（如果提供）
    // - 真实姓名：最多50字符（如果提供）
    
    // 5.2 检查用户名是否已存在
    var existUser models.User
    err := database.DB.Where("username = ?", req.Username).First(&existUser).Error
    if err == nil {
        return nil, errors.New("用户名已存在")
    }
    if err != gorm.ErrRecordNotFound {
        return nil, errors.New("数据库查询失败")
    }
    
    // 5.3 检查邮箱是否已被使用（如果提供了邮箱）
    if req.Email != "" {
        err = database.DB.Where("email = ? AND email != ''", req.Email).First(&existUser).Error
        if err == nil {
            return nil, errors.New("邮箱已被使用")
        }
        if err != gorm.ErrRecordNotFound {
            return nil, errors.New("数据库查询失败")
        }
    }
    
    // 5.4 检查手机号是否已被使用（如果提供了手机号）
    if req.Phone != "" {
        err = database.DB.Where("phone = ? AND phone != ''", req.Phone).First(&existUser).Error
        if err == nil {
            return nil, errors.New("手机号已被使用")
        }
        if err != gorm.ErrRecordNotFound {
            return nil, errors.New("数据库查询失败")
        }
    }
    
    // 5.5 加密密码
    hashedPassword, err := utils.HashPassword(req.Password)
    if err != nil {
        return nil, errors.New("密码加密失败")
    }
    // 使用bcrypt算法加密，生成类似：
    // $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
    
    // 5.6 创建用户对象
    user := &models.User{
        Username: strings.TrimSpace(req.Username),
        Password: hashedPassword,
        Email:    strings.TrimSpace(req.Email),
        Phone:    strings.TrimSpace(req.Phone),
        RealName: strings.TrimSpace(req.RealName),
        Role:     models.RoleCustomer, // 默认为客户（1）
        Status:   1,                   // 默认启用
    }
    
    // 5.7 保存到数据库
    if err := database.DB.Create(user).Error; err != nil {
        return nil, errors.New("用户创建失败")
    }
    // GORM会自动：
    // - 生成ID（自增主键）
    // - 设置CTime（创建时间戳）
    // - 设置MTime（修改时间戳）
    
    // 5.8 清除密码字段，避免返回
    user.Password = ""
    
    return user, nil
}
```

**执行的SQL查询**:
```sql
-- 5.2 检查用户名
SELECT * FROM users WHERE username = 'zhangsan' LIMIT 1;

-- 5.3 检查邮箱（如果提供）
SELECT * FROM users WHERE email = 'zhangsan@example.com' AND email != '' LIMIT 1;

-- 5.4 检查手机号（如果提供）
SELECT * FROM users WHERE phone = '13800138000' AND phone != '' LIMIT 1;

-- 5.7 插入用户
INSERT INTO users (username, password, email, phone, real_name, role, status, ctime, mtime)
VALUES ('zhangsan', '$2a$10$...', 'zhangsan@example.com', '13800138000', '张三', 1, 1, 1709125657, 1709125657);
```

### 第6步：密码加密

**文件**: `utils/password.go`

```go
// HashPassword 加密密码
func HashPassword(password string) (string, error) {
    // 使用bcrypt算法，cost=10
    bytes, err := bcrypt.GenerateFromPassword([]byte(password), 10)
    if err != nil {
        return "", err
    }
    return string(bytes), nil
}
```

**加密过程**:
```
原始密码: "123456"
    ↓
bcrypt算法（cost=10，自动加盐）
    ↓
加密结果: "$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy"
```

**特点**:
- 每次加密结果都不同（随机盐）
- 不可逆（无法解密）
- 验证时使用 `bcrypt.CompareHashAndPassword()`

### 第7步：数据库操作

**文件**: `database/database.go`

```go
// GORM执行Create操作

```

**GORM自动处理**:
1. 构建INSERT SQL语句
2. 参数化查询（防止SQL注入）
3. 执行SQL
4. 获取自增ID并赋值给user.ID
5. 自动填充时间戳字段

**生成的SQL**:
```sql
INSERT INTO `users` 
(`username`, `password`, `email`, `phone`, `real_name`, `role`, `status`, `ctime`, `mtime`) 
VALUES 
('zhangsan', '$2a$10$...', 'zhangsan@example.com', '13800138000', '张三', 1, 1, 1709125657, 1709125657);
```

**数据库表结构** (`models/user.go`):
```go
type User struct {
    ID       uint     `gorm:"primarykey;autoIncrement"`
    Username string   `gorm:"uniqueIndex;size:50;not null"`
    Password string   `gorm:"size:255;not null"`
    Email    string   `gorm:"size:100"`
    Phone    string   `gorm:"size:20"`
    RealName string   `gorm:"size:50"`
    Role     UserRole `gorm:"type:int;not null;default:1"`
    Status   int      `gorm:"default:1"`
    CTime    int64    `gorm:"autoCreateTime;not null"`
    MTime    int64    `gorm:"autoUpdateTime;not null"`
}
```

## 性能优化点

1. **数据库索引**
   - `username` 字段有唯一索引 (`uniqueIndex`)
   - 查询用户名时使用索引，速度快

2. **密码加密**
   - bcrypt cost=10，平衡安全性和性能
   - 加密操作在服务层，不阻塞其他请求

3. **参数验证**
   - 两层验证：Gin binding + 服务层
   - 早期失败，减少不必要的数据库查询

4. **GORM优化**
   - 使用参数化查询，防止SQL注入
   - 自动管理连接池

---

## 安全措施

1. **密码安全**
   - bcrypt加密，不可逆
   - 自动加盐，每次结果不同
   - 响应中不返回密码

2. **SQL注入防护**
   - GORM参数化查询
   - 不直接拼接SQL

3. **输入验证**
   - 用户名格式限制
   - 邮箱格式验证
   - 手机号格式验证
   - 长度限制

4. **唯一性约束**
   - 数据库级别的唯一索引
   - 应用层的重复检查

---

## 总结

用户注册的执行流程遵循标准的MVC架构模式：

1. **请求接收**: HTTP → 中间件 → 路由
2. **参数处理**: 控制器绑定和验证
3. **业务逻辑**: 服务层处理核心逻辑
4. **数据持久化**: GORM操作数据库
5. **响应返回**: 统一格式的JSON响应

整个流程清晰、安全、可维护。
