# 车辆与任务管理功能说明

## 功能概述

运输管理模块实现了车辆管理和运输任务管理的完整功能，包括车辆信息维护、运输任务创建与跟踪、车辆调度分配等核心业务功能。

## 主要功能

### 1. 车辆管理

#### 1.1 车辆信息管理
- **创建车辆**: 支持录入车牌号、车辆类型、载重量、司机等信息
- **查询车辆**: 支持按车牌号、车辆类型、状态、司机等条件筛选
- **更新车辆**: 支持修改车辆基本信息和司机分配
- **删除车辆**: 支持删除未使用的车辆（有运输任务的车辆不能删除）

#### 1.2 车辆状态管理
- **可用状态**: 车辆正常可用于运输任务
- **维修状态**: 车辆维修中，不可分配任务
- **状态切换**: 支持在可用和维修状态间切换

### 2. 运输任务管理

#### 2.1 任务创建与分配
- **手动创建**: 指定订单、车辆、司机创建运输任务
- **批量分配**: 支持将多个订单批量分配给同一车辆和司机
- **任务编号**: 自动生成唯一的任务编号（格式：T+时间戳）

#### 2.2 任务状态管理
- **待执行**: 任务创建后的初始状态
- **执行中**: 任务开始执行，记录开始时间
- **已完成**: 任务完成，记录结束时间
- **已取消**: 任务取消

#### 2.3 装卸扫描
- **装车扫描**: 根据运输任务ID和扫描码校验任务，记录装车站点与装车时间，并将任务推进为执行中
- **卸车扫描**: 根据运输任务ID和扫描码校验任务，记录卸车站点与卸车时间，并将任务标记为已完成
- **状态联动**: 装车扫描会在符合状态机规则时推动订单进入运输中，卸车扫描会在符合规则时推动订单进入目的地分拣

#### 2.4 装卸记录管理
- **列表查询**: 支持按任务编号、订单号、车辆、司机、站点、扫描类型、时间范围分页查询装卸记录
- **详情查看**: 支持根据记录ID查看单条装卸记录详情，包含任务、订单、站点、车辆、司机等关联信息

#### 2.5 任务信息维护
- **基本信息**: 起点、终点、距离、成本等
- **关联信息**: 订单、车辆、司机信息
- **时间记录**: 创建时间、开始时间、结束时间
- **备注信息**: 支持添加任务备注

#### 2.6 运输监控与异常预警
- **监控概览**: 统计待执行、执行中、已完成、已取消、延迟、异常等运输任务数量
- **任务监控**: 支持按任务编号、订单号、车辆、司机、状态、预警级别查询运输监控任务列表，查看进度、最新扫描、预计时长、实际时长、延迟时长等信息
- **异常预警**: 对订单异常、运输延迟、长时间未发车、高成本等场景生成预警列表，便于调度员和管理员及时处置

#### 2.7 运输成本核算
- **成本概览**: 汇总运输总成本、总里程、单任务平均成本、单公里平均成本、赔付成本、高成本任务数量等指标
- **任务成本列表**: 支持按任务、订单、车辆、司机、状态、成本区间查询成本核算结果
- **成本详情**: 查看单任务运输成本、单公里成本、预计时长、实际时长、装卸次数、赔付金额、最新扫描节点等信息

### 3. 统计分析

#### 3.1 车辆统计
- 总车辆数、可用车辆数、维修车辆数
- 总载重量、平均载重量

#### 3.2 任务统计  
- 各状态任务数量统计
- 总运输距离、总运输成本
- 任务状态分布占比

#### 3.3 司机统计
- 司机任务数量排行
- 司机运输距离和成本统计

## 业务规则

### 车辆管理规则
1. 车牌号必须唯一，不能重复
2. 司机必须具有司机权限（role >= 3）
3. 有运输任务的车辆不能删除，建议设为维修状态
4. 维修中的车辆不能分配新的运输任务

### 运输任务规则
1. 每个订单只能有一个有效的运输任务（非取消状态）
2. 只有可用状态的车辆才能分配任务
3. 司机必须具有相应权限
4. 任务状态转换规则：
   - 待执行 → 执行中、已取消
   - 执行中 → 已完成、已取消
   - 已完成、已取消 → 不能转换

### 数据完整性
1. 车辆删除前检查是否有关联的运输任务
2. 任务创建时验证订单、车辆、司机的有效性
3. 状态更新时验证转换的合法性
