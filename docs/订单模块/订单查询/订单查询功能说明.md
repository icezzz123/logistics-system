# 订单查询功能说明

## 功能概述

订单查询功能提供了强大的订单检索能力，支持分页、多条件搜索和筛选，帮助用户快速找到目标订单。

## 功能特性

### 1. 订单列表查询

#### 1.1 分页功能
- **默认分页**：第1页，每页10条
- **最大限制**：每页最多100条记录
- **分页信息**：返回总记录数、总页数、当前页码

#### 1.2 搜索功能
- **订单号搜索**：支持模糊匹配
- **示例**：输入 "ORD20260304" 可以匹配所有该日期的订单

#### 1.3 筛选功能

| 筛选条件 | 说明 | 示例 |
|---------|------|------|
| 订单状态 | 按订单状态筛选 | status=1（待处理） |
| 发件国家 | 按发件国家筛选 | sender_country=中国 |
| 收件国家 | 按收件国家筛选 | receiver_country=美国 |
| 时间范围 | 按订单创建时间筛选 | start_time=1772617000&end_time=1772620000 |

#### 1.4 排序规则
- 默认按订单创建时间倒序排列（最新的在前）

#### 1.5 权限控制
- **普通客户**：只能查看自己创建的订单
- **调度员**：可以查看所有订单
- **管理员**：可以查看所有订单

### 2. 订单详情查询

#### 2.1 功能说明
- 根据订单ID查询完整的订单信息
- 包含所有订单字段（发件人、收件人、货物、运输、费用等）

#### 2.2 权限控制
- **普通客户**：只能查看自己的订单详情
- **调度员**：可以查看任何订单详情
- **管理员**：可以查看任何订单详情

## API接口

### 接口1：订单列表查询

#### 请求地址
```
GET /api/orders
```

#### 请求头
```
Authorization: Bearer {token}
```

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|-------|------|------|------|------|
| page | int | 否 | 页码，默认1 | 1 |
| page_size | int | 否 | 每页数量，默认10，最大100 | 10 |
| order_no | string | 否 | 订单号（模糊搜索） | ORD20260304 |
| status | int | 否 | 订单状态 | 1 |
| sender_country | string | 否 | 发件国家 | 中国 |
| receiver_country | string | 否 | 收件国家 | 美国 |
| start_time | int64 | 否 | 开始时间（Unix时间戳） | 1772617000 |
| end_time | int64 | 否 | 结束时间（Unix时间戳） | 1772620000 |

#### 请求示例

**示例1：基本查询**
```
GET /api/orders?page=1&page_size=10
```

**示例2：按订单号搜索**
```
GET /api/orders?order_no=ORD20260304174355
```

**示例3：按国家筛选**
```
GET /api/orders?receiver_country=美国
```

**示例4：按状态筛选**
```
GET /api/orders?status=1
```

**示例5：按时间范围筛选**
```
GET /api/orders?start_time=1772617000&end_time=1772620000
```

**示例6：组合查询**
```
GET /api/orders?receiver_country=美国&status=1&page=1&page_size=20
```

## 使用场景

### 场景1：客户查看自己的订单
```
用户登录 → 访问订单列表 → 查看自己的所有订单
```

### 场景2：按订单号搜索
```
用户输入订单号 → 系统模糊匹配 → 返回匹配的订单列表
```

### 场景3：筛选特定国家的订单
```
选择收件国家 → 系统筛选 → 返回该国家的所有订单
```

### 场景4：查看订单详情
```
点击订单 → 查看完整信息 → 包含所有字段
```

### 场景5：管理员查看所有订单
```
管理员登录 → 访问订单列表 → 查看所有用户的订单
```

### 场景6：按时间范围查询
```
选择开始和结束时间 → 系统筛选 → 返回时间范围内的订单
```

## 错误处理

### 常见错误

1. **未登录**
```json
{
  "code": 401,
  "message": "未登录"
}
```

2. **无权查看**
```json
{
  "code": 403,
  "message": "无权查看此订单"
}
```

3. **订单不存在**
```json
{
  "code": 404,
  "message": "订单不存在"
}
```

4. **参数错误**
```json
{
  "code": 400,
  "message": "参数错误: 无效的订单ID"
}
```

## 性能优化

1. **分页限制**：最大每页100条，避免一次查询过多数据
2. **索引优化**：订单号、状态、客户ID、时间字段都有索引
3. **字段精简**：列表查询只返回必要字段，减少数据传输
4. **权限过滤**：在数据库层面进行权限过滤，提高效率

## 注意事项

1. 时间参数使用Unix时间戳（秒）
2. 订单号搜索支持模糊匹配，会自动添加通配符
3. 普通用户只能查看自己的订单，不能查看其他用户的订单
4. 分页从第1页开始，不是第0页
5. 订单列表按创建时间倒序排列，最新的在前
6. 所有查询都需要提供有效的认证令牌
7. 时间格式统一为 "2006-01-02 15:04:05"
