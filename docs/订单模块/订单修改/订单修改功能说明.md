# 订单修改功能说明

## 功能概述

订单修改功能允许用户在订单发货前修改收件人信息和备注，同时提供订单取消功能，确保订单信息的准确性和灵活性。

## 功能特性

### 1. 订单信息修改

#### 1.1 可修改字段
- **收件人姓名**：修改收件人的姓名
- **收件人电话**：修改收件人的联系电话
- **收件人省份**：修改收件人所在省份
- **收件人城市**：修改收件人所在城市
- **收件人地址**：修改收件人的详细地址
- **收件人邮编**：修改收件人的邮政编码
- **订单备注**：添加或修改订单备注信息

#### 1.2 不可修改字段
以下字段一旦订单创建后不可修改：
- 订单号
- 发件人信息（姓名、电话、地址等）
- 货物信息（名称、重量、体积等）
- 运输方式
- 运费和保险费
- 订单创建时间

#### 1.3 修改限制条件

**状态限制：**
- 只有**待处理（status=1）**状态的订单可以修改
- 已接单、运输中、已完成等状态的订单不能修改

**权限限制：**
- **普通用户**：只能修改自己创建的订单
- **调度员**：可以修改任何订单
- **管理员**：可以修改任何订单

**字段限制：**
- 至少需要提供一个要修改的字段
- 所有字段都是可选的，只修改提供的字段

### 2. 订单取消功能

#### 2.1 可取消状态
- **待处理（status=1）**：订单刚创建，还未处理
- **已接单（status=2）**：订单已被接单，但还未入库

#### 2.2 不可取消状态
以下状态的订单不能取消：
- 已入库（status=3）及之后的所有状态
- 已取消（status=12）的订单

#### 2.3 取消后状态
- 订单状态变更为**已取消（status=12）**
- 已取消的订单不能再修改
- 已取消的订单不能恢复

#### 2.4 权限控制
- **普通用户**：只能取消自己的订单
- **调度员**：可以取消任何订单
- **管理员**：可以取消任何订单

## API接口

### 接口1：修改订单信息

#### 请求地址
```
PUT /api/orders/{id}
```

#### 请求头
```
Authorization: Bearer {token}
Content-Type: application/json
```

#### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| id | int | 是 | 订单ID |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 最大长度 |
|-------|------|------|------|---------|
| receiver_name | string | 否 | 收件人姓名 | 50 |
| receiver_phone | string | 否 | 收件人电话 | 20 |
| receiver_province | string | 否 | 收件人省份 | 50 |
| receiver_city | string | 否 | 收件人城市 | 50 |
| receiver_address | string | 否 | 收件人地址 | 255 |
| receiver_postcode | string | 否 | 收件人邮编 | 20 |
| remark | string | 否 | 订单备注 | - |

**注意：**
- 所有字段都是可选的
- 至少需要提供一个字段
- 只会更新提供的字段，未提供的字段保持不变

#### 请求示例

**示例1：修改收件人姓名和电话**
```json
{
  "receiver_name": "田中太郎",
  "receiver_phone": "+81-90-1234-5678"
}
```

**示例2：修改收件人地址**
```json
{
  "receiver_address": "東京都渋谷区1-2-3",
  "receiver_city": "東京",
  "receiver_postcode": "150-0001"
}
```

**示例3：添加备注**
```json
{
  "remark": "请在工作日配送，周末不在家"
}
```

**示例4：修改多个字段**
```json
{
  "receiver_name": "张三",
  "receiver_phone": "13800138000",
  "receiver_city": "北京",
  "receiver_address": "朝阳区建国路1号",
  "remark": "请提前电话联系"
}
```

#### 响应结果

**成功响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "message": "订单修改成功"
  }
}
```

**错误响应：**

1. 订单不存在
```json
{
  "code": 404,
  "message": "订单不存在"
}
```

2. 无权修改
```json
{
  "code": 500,
  "message": "无权修改此订单"
}
```

3. 状态不允许修改
```json
{
  "code": 500,
  "message": "只有待处理状态的订单可以修改"
}
```

4. 没有要更新的字段
```json
{
  "code": 500,
  "message": "没有要更新的字段"
}
```

## 使用场景

### 场景1：修改收件人信息
```
用户下单后发现收件人信息填写错误
→ 在订单待处理状态时修改收件人信息
→ 系统验证权限和状态
→ 更新订单信息
→ 返回修改成功
```

### 场景2：添加配送备注
```
用户想要添加配送要求
→ 修改订单备注字段
→ 添加"请在工作日配送"等信息
→ 系统保存备注
→ 快递员可以看到备注信息
```

### 场景3：取消订单
```
用户不需要此订单了
→ 在订单待处理或已接单状态时取消
→ 系统验证权限和状态
→ 订单状态变更为已取消
→ 返回取消成功
```

### 场景4：尝试修改已发货订单（失败）
```
用户尝试修改已发货的订单
→ 系统检查订单状态
→ 发现订单已不是待处理状态
→ 返回错误："只有待处理状态的订单可以修改"
```

### 场景5：管理员修改订单
```
客户联系客服要求修改订单
→ 管理员登录系统
→ 查找客户订单
→ 修改收件人信息
→ 系统验证管理员权限
→ 修改成功
```

## 业务规则

### 1. 修改规则
1. 只能修改待处理状态的订单
2. 用户只能修改自己的订单（管理员除外）
3. 至少需要提供一个要修改的字段
4. 所有字符串字段会自动去除首尾空格
5. 修改后订单的修改时间（mtime）会自动更新

### 2. 取消规则
1. 只能取消待处理和已接单状态的订单
2. 用户只能取消自己的订单（管理员除外）
3. 取消后订单状态变为已取消（status=12）
4. 已取消的订单不能恢复
5. 已取消的订单不能再修改

### 3. 权限规则
- **客户（role=1）**：只能操作自己的订单
- **快递员（role=2）**：只能操作自己的订单
- **分拣员（role=3）**：只能操作自己的订单
- **司机（role=4）**：只能操作自己的订单
- **站点管理员（role=5）**：只能操作自己的订单
- **调度员（role=6）**：可以操作所有订单
- **管理员（role=7）**：可以操作所有订单

## 状态流转图

```
待处理(1) ──修改信息──> 待处理(1)
    │
    ├──取消订单──> 已取消(12)
    │
    └──接单──> 已接单(2)
                  │
                  ├──取消订单──> 已取消(12)
                  │
                  └──入库──> 已入库(3) [不可修改、不可取消]
```

## 错误处理

### 常见错误及解决方案

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| 订单不存在 | 订单ID不正确 | 检查订单ID是否正确 |
| 无权修改此订单 | 尝试修改其他用户的订单 | 只能修改自己的订单 |
| 只有待处理状态的订单可以修改 | 订单已发货或已完成 | 联系客服处理 |
| 没有要更新的字段 | 请求体为空或所有字段为空 | 至少提供一个字段 |
| 无权取消此订单 | 尝试取消其他用户的订单 | 只能取消自己的订单 |
| 当前状态的订单不能取消 | 订单已入库或更后面的状态 | 联系客服处理 |

## 注意事项

1. **修改时机**：建议在订单创建后尽快修改，一旦订单被接单处理，修改的机会就很有限
2. **字段验证**：虽然所有字段都是可选的，但提供的字段会进行格式验证
3. **权限检查**：每次修改都会检查用户权限，确保安全性
4. **状态检查**：每次修改都会检查订单状态，确保业务逻辑正确
5. **数据持久化**：修改后数据立即保存到数据库
6. **时间更新**：修改后订单的修改时间会自动更新
7. **取消不可逆**：订单一旦取消，不能恢复，请谨慎操作
8. **核心字段保护**：发件人信息、货物信息、运费等核心字段不能修改

## 最佳实践

### 1. 修改订单
```javascript
// 前端示例代码
async function updateOrder(orderId, updateData) {
  try {
    const response = await fetch(`/api/orders/${orderId}`, {
      method: 'PUT',
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(updateData)
    });
    
    const result = await response.json();
    
    if (result.code === 200) {
      alert('订单修改成功');
      // 刷新订单详情
      loadOrderDetail(orderId);
    } else {
      alert(`修改失败：${result.message}`);
    }
  } catch (error) {
    console.error('修改订单失败', error);
  }
}
```

### 2. 取消订单
```javascript
// 前端示例代码
async function cancelOrder(orderId) {
  if (!confirm('确定要取消此订单吗？取消后不可恢复！')) {
    return;
  }
  
  try {
    const response = await fetch(`/api/orders/${orderId}/cancel`, {
      method: 'PUT',
      headers: {
        'Authorization': `Bearer ${token}`
      }
    });
    
    const result = await response.json();
    
    if (result.code === 200) {
      alert('订单已取消');
      // 返回订单列表
      window.location.href = '/orders';
    } else {
      alert(`取消失败：${result.message}`);
    }
  } catch (error) {
    console.error('取消订单失败', error);
  }
}
```
