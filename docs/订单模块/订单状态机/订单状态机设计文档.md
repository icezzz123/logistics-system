# 订单状态机设计文档

## 概述

订单状态机用于管理订单在整个物流流程中的状态流转，确保状态转换的合法性和可追溯性。

## 订单状态定义

| 状态码 | 状态名称 | 说明 | 类型 |
|--------|----------|------|------|
| 1 | 待处理 | 订单已创建，等待处理 | 初始状态 |
| 2 | 已接单 | 调度员已接单 | 中间状态 |
| 3 | 已入库 | 货物已入库 | 中间状态 |
| 4 | 分拣中 | 正在进行分拣 | 中间状态 |
| 5 | 运输中 | 正在运输途中 | 中间状态 |
| 6 | 清关中 | 正在进行清关（跨境订单） | 中间状态 |
| 7 | 目的地分拣 | 到达目的地，正在分拣 | 中间状态 |
| 8 | 配送中 | 快递员正在配送 | 中间状态 |
| 9 | 已送达 | 已送达收件地址 | 中间状态 |
| 10 | 已签收 | 收件人已签收 | 终态 |
| 11 | 异常 | 订单出现异常 | 特殊状态 |
| 12 | 已取消 | 订单已取消 | 终态 |

## 状态流转规则

### 标准流程

```
待处理(1) → 已接单(2) → 已入库(3) → 分拣中(4) → 运输中(5) 
  → 清关中(6) → 目的地分拣(7) → 配送中(8) → 已送达(9) → 已签收(10)
```

**说明：**
- 清关中(6)仅适用于跨境订单，国内订单可跳过
- 运输中(5)可直接到目的地分拣(7)

### 取消流程

```
待处理(1) → 已取消(12)
已接单(2) → 已取消(12)
```

**说明：**
- 只有待处理和已接单状态可以取消
- 客户、调度员、管理员可以取消订单

### 异常处理流程

```
任何中间状态 → 异常(11)
异常(11) → 恢复到之前的状态
```

**说明：**
- 除已取消和已签收外，任何状态都可能出现异常
- 异常状态可由调度员或管理员恢复到正常状态

## 状态转换权限

### 角色定义

| 角色ID | 角色名称 | 说明 |
|--------|----------|------|
| 1 | 客户 | 普通用户 |
| 2 | 快递员 | 负责配送 |
| 3 | 分拣员 | 负责分拣 |
| 4 | 司机 | 负责运输 |
| 5 | 站点管理员 | 管理站点 |
| 6 | 调度员 | 调度管理 |
| 7 | 管理员 | 系统管理员 |

### 转换权限表

| 从状态 | 到状态 | 允许的角色 |
|--------|--------|------------|
| 待处理 | 已接单 | 调度员、管理员 |
| 待处理 | 已取消 | 客户、调度员、管理员 |
| 已接单 | 已入库 | 站点管理员、调度员、管理员 |
| 已接单 | 已取消 | 客户、调度员、管理员 |
| 已入库 | 分拣中 | 分拣员、站点管理员、调度员、管理员 |
| 分拣中 | 运输中 | 司机、调度员、管理员 |
| 运输中 | 清关中 | 调度员、管理员 |
| 运输中 | 目的地分拣 | 调度员、管理员 |
| 清关中 | 目的地分拣 | 调度员、管理员 |
| 目的地分拣 | 配送中 | 快递员、调度员、管理员 |
| 配送中 | 已送达 | 快递员、调度员、管理员 |
| 已送达 | 已签收 | 快递员、调度员、管理员 |
| 任何中间状态 | 异常 | 快递员、分拣员、司机、站点管理员、调度员、管理员 |
| 异常 | 任何中间状态 | 调度员、管理员 |

## 状态机实现

### 核心功能

1. **CanTransition** - 检查是否可以进行状态转换
   - 验证状态转换规则
   - 验证用户角色权限
   - 检查终态保护

2. **GetAllowedTransitions** - 获取当前状态允许的所有转换
   - 根据当前状态和用户角色
   - 返回可转换的状态列表

3. **ValidateTransition** - 验证状态转换并返回详细错误
   - 状态相同检查
   - 终态检查
   - 转换规则检查
   - 权限检查

4. **GetStatusFlow** - 获取订单的标准流程
   - 返回标准状态流转顺序

### 终态保护

**终态定义：**
- 已签收(10)
- 已取消(12)

**保护规则：**
- 终态不能转换到任何其他状态
- 确保订单完结后的数据稳定性

## API接口

### 1. 更新订单状态

```
PUT /api/orders/:id/status
```

**请求体：**
```json
{
  "status": 2,
  "remark": "备注信息（可选）"
}
```

**响应：**
```json
{
  "code": 200,
  "message": "success"
}
```

**错误响应：**
```json
{
  "code": 500,
  "message": "不允许的状态转换"
}
```

### 2. 获取允许的状态转换

```
GET /api/orders/:id/transitions
```

**响应：**
```json
{
  "code": 200,
  "data": {
    "current_status": 1,
    "current_status_name": "待处理",
    "allowed_statuses": [
      {
        "status": 2,
        "status_name": "已接单"
      },
      {
        "status": 12,
        "status_name": "已取消"
      }
    ]
  }
}
```

## 使用示例

### 示例1：正常流程

```
1. 客户创建订单 → 待处理(1)
2. 调度员接单 → 已接单(2)
3. 站点入库 → 已入库(3)
4. 分拣员分拣 → 分拣中(4)
5. 司机装车 → 运输中(5)
6. 到达目的地 → 目的地分拣(7)
7. 快递员配送 → 配送中(8)
8. 送达地址 → 已送达(9)
9. 客户签收 → 已签收(10)
```

### 示例2：跨境订单

```
1. 待处理(1) → 已接单(2) → 已入库(3) → 分拣中(4)
2. 运输中(5) → 清关中(6) → 目的地分拣(7)
3. 配送中(8) → 已送达(9) → 已签收(10)
```

### 示例3：异常处理

```
1. 配送中(8) → 异常(11)  // 地址错误
2. 异常(11) → 配送中(8)  // 调度员修正地址后恢复
3. 配送中(8) → 已送达(9) → 已签收(10)
```

### 示例4：订单取消

```
1. 待处理(1) → 已取消(12)  // 客户取消
或
2. 已接单(2) → 已取消(12)  // 调度员取消
```

## 注意事项

1. **状态转换必须遵循规则** - 不能跳过中间状态
2. **终态不可修改** - 已签收和已取消是最终状态
3. **权限严格控制** - 不同角色只能执行特定的状态转换
4. **异常可恢复** - 异常状态可由管理人员恢复
5. **记录时间戳** - 关键状态转换时记录时间

## 扩展性

状态机设计支持以下扩展：

1. **添加新状态** - 在状态定义中添加新的中间状态
2. **修改转换规则** - 在stateTransitions中添加新的转换规则
3. **调整权限** - 修改AllowedBy字段调整角色权限
4. **状态日志** - 可扩展记录每次状态转换的日志
5. **通知机制** - 可在状态转换时触发通知
