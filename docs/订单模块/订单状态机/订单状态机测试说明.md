# 订单状态机测试说明

## 测试目的
验证订单状态机的终态保护功能是否正常工作。

## 测试环境
- 系统：跨境物流作业系统
- API地址：http://localhost:8080
- 测试工具：Postman（推荐）或curl

## 测试账号
需要使用管理员账号（角色=7）进行测试。

注意：admin001账号的角色是1（普通用户），不是管理员，无法进行完整测试。

## 代码验证

如果测试失败，需要检查以下代码：

1. `services/order_state_machine.go` - ValidateTransition函数
   - 检查终态保护逻辑（已签收、已取消）
   - 检查状态转换规则表

2. `services/order_service.go` - UpdateOrderStatus函数
   - 确认是否正确调用ValidateTransition
   - 确认错误是否正确返回

3. `controllers/order_controller.go` - UpdateOrderStatus控制器
   - 确认错误处理逻辑
   - 确认HTTP状态码返回

## 常见问题

### Q1: 测试时提示"无权执行此状态转换"
A: 检查使用的账号角色是否正确。某些状态转换只有特定角色才能执行。

### Q2: PowerShell测试时中文显示为乱码
A: 建议使用Postman进行测试，避免PowerShell的中文编码问题。

### Q3: 测试显示"修改成功"但实际未修改
A: 这可能是测试脚本的错误捕获问题。应该检查：
- HTTP响应状态码
- 响应body中的code字段
- 数据库中的实际数据

## 测试结论

终态保护功能的核心逻辑在`ValidateTransition`函数中：

```go
// 终态检查
if fromStatus == models.OrderCancelled {
    return errors.New("已取消的订单不能修改状态")
}
if fromStatus == models.OrderSigned {
    return errors.New("已签收的订单不能修改状态")
}
```

这段代码确保了已签收和已取消的订单不能再进行状态转换，是订单状态机的重要保护机制。
