## 数据库设计亮点

### 1. 跨境物流特性支持
- 订单表增加了发件国家、收件国家字段，支持跨境业务
- 站点表支持多国站点管理（country, province, city）
- 订单表增加海关信息字段（customs_declaration, hs_code）
- 支持多币种结算（currency字段）
- 支持多种运输方式（air, sea, land, rail, express）

### 2. 完整的订单生命周期管理
订单状态覆盖全流程：
```
pending → accepted → in_warehouse → sorting → in_transit 
→ customs_clearing → destination_sorting → delivering 
→ delivered → signed
```

### 3. 智能分拣系统设计
- 分拣规则表：支持按国家、省、市、区县配置路由规则
- 分拣任务表：支持任务分配和进度跟踪
- 分拣记录表：记录每次扫描操作，支持准确率统计

### 4. 全程追踪能力
- 追踪记录表：记录订单在各节点的状态
- 站点流转表：记录货物在站点间的流转
- 配送记录表：记录最后一公里配送信息
- 签收记录表：记录最终签收信息

### 5. 异常处理机制
- 异常类型：破损、丢失、延误、拒收、地址错误、海关扣留等
- 异常状态：待处理、处理中、已解决、已关闭
- 支持异常上报、处理、赔偿全流程

### 6. 多种签收方式
- 本人签收（self）
- 代收（proxy）- 需记录代收人信息
- 快递柜（locker）
- 驿站（station）
- 拒收（refused）

### 7. 索引优化策略
- 高频查询字段建立索引（status, country, city等）
- 外键字段统一建立索引
- 唯一性约束字段自动建立唯一索引
- 软删除字段（deleted_at）建立索引

### 8. 数据安全设计
- 密码字段使用bcrypt加密
- 敏感信息（身份证号）加密存储
- 软删除机制保护数据
- 支持数据审计（created_at, updated_at）

## 技术债务和注意事项

### 需要注意的点
1. 跨服务数据一致性：使用分布式事务或最终一致性方案
2. 外键约束：微服务架构下，跨库外键需要在应用层维护
3. 数据同步：订单状态变更需要同步到追踪服务
4. 缓存策略：订单状态、站点信息等热点数据需要缓存
5. 数据备份：制定备份策略，确保数据安全

### 性能优化建议
1. 订单表数据量大，考虑分表策略（按月或按年）
2. 追踪记录表增长快，考虑定期归档历史数据
3. 分拣记录表高频写入，考虑批量插入优化
4. 使用读写分离提升查询性能
5. 关键查询使用覆盖索引
