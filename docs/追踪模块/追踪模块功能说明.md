# 追踪模块功能说明

## 功能概述
追踪模块用于记录订单在物流作业过程中的追踪节点信息，支持追踪记录创建、列表查询、详情查询、按订单查看历史、时间轴展示以及基础时效/延误预警。

---

## 已实现功能

### 1. 追踪记录创建
- 支持按订单创建追踪记录
- 支持记录位置、经纬度、状态、描述
- 支持指定追踪时间，便于补录历史节点
- 自动记录操作人

### 2. 追踪记录查询
- 支持追踪记录分页查询
- 支持按订单ID、订单号、状态、操作人、时间范围筛选
- 支持单条记录详情查询

### 3. 订单追踪历史
- 支持按订单获取完整追踪历史
- 返回订单当前状态与追踪记录列表
- 适合后台列表和客服查询场景

### 4. 追踪时间轴
- 支持按订单生成时间轴数据
- 按时间正序输出追踪节点
- 返回最新位置、最新更新时间、预计送达时间、是否延误等信息

### 5. 时效/延误预警
当前已实现基础规则：
- **延误预警**: 当前时间已超过 `order_time + estimated_days`
- **时效预警**: 距离预计送达时间不足 24 小时
- **长时间无更新预警**: 运输中、清关中、目的地分拣、配送中状态下，超过 48 小时无追踪更新

---

## API接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/tracking/records | 创建追踪记录 |
| GET | /api/tracking/records | 获取追踪记录列表 |
| GET | /api/tracking/records/:id | 获取追踪记录详情 |
| GET | /api/tracking/orders/:order_id/history | 获取订单追踪历史 |
| GET | /api/tracking/orders/:order_id/timeline | 获取订单追踪时间轴 |
| GET | /api/tracking/warnings | 获取追踪预警列表 |

---

## 权限说明
- 查询类接口要求具备 `tracking:view`
- 创建接口要求具备 `tracking:update`
- 客户角色仅允许查看自己的订单追踪信息
- 员工、站点管理员、调度员、管理员可查看全部符合权限的数据

---

## 关键字段说明

### 创建追踪记录请求示例
```json
{
  "order_id": 31,
  "location": "上海转运中心",
  "latitude": 31.2304,
  "longitude": 121.4737,
  "status": "已到达转运中心",
  "description": "包裹已完成转运扫描",
  "track_time": 1774024744
}
```

### 追踪预警字段说明
- `warning_level`: 预警级别，`warning` 或 `critical`
- `warning_type`: 预警类型，`timeliness`、`delay`、`stale_update`
- `remaining_hours`: 距预计送达剩余小时数
- `overdue_hours`: 已超时小时数
- `latest_track_time`: 最新追踪更新时间

---

## 使用建议
- 追踪记录建议在关键节点录入，例如到站、装车、出库、派送、签收。
- 时间轴适合前端直接展示物流轨迹。
- 预警接口适合调度台、客服或异常监控大屏使用。
