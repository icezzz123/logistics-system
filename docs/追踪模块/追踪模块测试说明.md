# 追踪模块测试说明

## 测试目标
验证追踪模块的以下能力：
- 追踪记录创建
- 追踪记录列表查询
- 追踪记录详情查询
- 订单追踪历史查询
- 订单追踪时间轴查询
- 追踪预警查询

---

## 前置条件
- 服务已启动
- 已存在可登录账号
- 已存在测试订单
- 请求头中已带有效 JWT Token

---

## 推荐测试用例

### 用例1：创建追踪记录
**请求**: `POST /api/tracking/records`

**验证点**:
- 返回记录ID
- 返回订单号、位置、状态、追踪时间
- 操作人字段正确

### 用例2：查询追踪记录列表
**请求**: `GET /api/tracking/records?page=1&page_size=10&order_no=xxx`

**验证点**:
- 支持分页
- 支持按订单号筛选
- 返回总数、页码、列表项

### 用例3：查询追踪记录详情
**请求**: `GET /api/tracking/records/:id`

**验证点**:
- 返回单条记录完整信息
- 记录ID与请求一致

### 用例4：查询订单追踪历史
**请求**: `GET /api/tracking/orders/:order_id/history`

**验证点**:
- 返回订单信息与追踪记录列表
- 历史记录按时间倒序返回

### 用例5：查询订单追踪时间轴
**请求**: `GET /api/tracking/orders/:order_id/timeline`

**验证点**:
- 时间轴按时间正序返回
- 返回最新位置、最新更新时间、预计送达时间
- 若超时则 `is_delayed = true`

### 用例6：查询追踪预警
**请求**: `GET /api/tracking/warnings?page=1&page_size=10`

**验证点**:
- 可返回 warning / critical 级别预警
- 支持按 `warning_level` 和 `warning_type` 过滤
- 返回剩余时效或超时时长信息

## PowerShell联调示例

### 1. 登录获取 Token
```powershell
$loginBody = @{
    username = 'admin'
    password = 'Pass1234'
} | ConvertTo-Json

$loginResp = Invoke-RestMethod -Method Post `
    -Uri 'http://127.0.0.1:8080/api/auth/login' `
    -ContentType 'application/json' `
    -Body $loginBody

$token = $loginResp.data.token
$headers = @{ Authorization = "Bearer $token" }
```

### 2. 创建追踪记录
```powershell
$body = @{
    order_id     = 31
    location     = '上海转运中心'
    latitude     = 31.2304
    longitude    = 121.4737
    status       = '已到达转运中心'
    description  = '包裹已完成转运扫描'
    track_time   = [int64]([DateTimeOffset]::Now.ToUnixTimeSeconds())
} | ConvertTo-Json

Invoke-RestMethod -Method Post `
    -Uri 'http://127.0.0.1:8080/api/tracking/records' `
    -Headers $headers `
    -ContentType 'application/json' `
    -Body $body
```

### 3. 查询追踪记录列表
```powershell
Invoke-RestMethod -Method Get `
    -Uri 'http://127.0.0.1:8080/api/tracking/records?page=1&page_size=10&order_no=ORD202603210039040021048' `
    -Headers $headers
```

### 4. 查询追踪记录详情
```powershell
Invoke-RestMethod -Method Get `
    -Uri 'http://127.0.0.1:8080/api/tracking/records/1' `
    -Headers $headers
```

### 5. 查询订单追踪历史
```powershell
Invoke-RestMethod -Method Get `
    -Uri 'http://127.0.0.1:8080/api/tracking/orders/31/history' `
    -Headers $headers
```

### 6. 查询订单追踪时间轴
```powershell
Invoke-RestMethod -Method Get `
    -Uri 'http://127.0.0.1:8080/api/tracking/orders/31/timeline' `
    -Headers $headers
```

### 7. 查询追踪预警
```powershell
Invoke-RestMethod -Method Get `
    -Uri 'http://127.0.0.1:8080/api/tracking/warnings?page=1&page_size=10&warning_level=warning' `
    -Headers $headers
```

---

## 当前验证建议
- 至少创建 2 条同一订单的追踪记录，用于验证历史与时间轴顺序。
- 至少准备 1 条运输中且长期无更新的订单，用于验证 `stale_update` 预警。
- 如需验证 `delay` 预警，可准备已超过预计送达时间的订单。
