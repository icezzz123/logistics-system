# 库存预警功能说明

## 功能概述

库存预警功能用于监控各站点的库存使用情况，当站点的订单数量达到预设阈值时，自动触发预警，帮助管理人员及时发现库存压力，采取相应措施。

## 业务说明

库存预警是物流管理的重要工具，通过实时监控站点容量使用率，可以：
- 及时发现库存压力，避免爆仓
- 优化货物分配，平衡各站点负载
- 提前安排货物转移，确保运营顺畅
- 为扩容决策提供数据支持

## 预警级别

系统根据容量使用率自动判断预警级别：

| 预警级别 | 使用率范围 | 颜色标识 | 说明 |
|---------|-----------|---------|------|
| normal（正常） | < 70% | 绿色 | 库存正常，继续保持 |
| warning（警告） | 70% ≤ 使用率 < 90% | 黄色 | 使用率较高，需要关注 |
| critical（严重） | ≥ 90% | 红色 | 库存严重超载，需要立即处理 |

## API接口

### 1. 获取库存预警信息

**接口地址**: `GET /api/stations/inventory/warnings`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| station_id | uint | 否 | 站点ID，0或不传表示查询所有站点 |
| warning_level | int | 否 | 预警级别：0=所有，1=warning，2=critical |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "summary": {
      "total_stations": 4,
      "normal_stations": 3,
      "warning_stations": 1,
      "critical_stations": 0
    },
    "warnings": [
      {
        "station_id": 5,
        "station_name": "广州目的站",
        "station_code": "GZ-DEST-001",
        "current_count": 4,
        "capacity": 5,
        "usage_rate": 0.8,
        "usage_percent": "80.0%",
        "warning_level": "warning",
        "warning_message": "库存使用率较高，当前80.0%，建议关注",
        "last_check_time": 1773157800,
        "recommend_action": "准备清理库存，加快货物流转速度"
      }
    ]
  }
}
```

### 2. 按预警级别获取站点列表

**接口地址**: `GET /api/stations/inventory/warnings/{level}`

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| level | string | 是 | 预警级别：warning, critical, all |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "level": "warning",
    "count": 1,
    "warnings": [
      {
        "station_id": 5,
        "station_name": "广州目的站",
        "current_count": 4,
        "capacity": 5,
        "usage_percent": "80.0%",
        "warning_level": "warning",
        "warning_message": "库存使用率较高，当前80.0%，建议关注",
        "recommend_action": "准备清理库存，加快货物流转速度"
      }
    ]
  }
}
```

## 响应字段说明

### 预警汇总字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| total_stations | int | 总站点数 |
| normal_stations | int | 正常状态站点数 |
| warning_stations | int | 警告状态站点数 |
| critical_stations | int | 严重预警站点数 |

### 预警详情字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| station_id | uint | 站点ID |
| station_name | string | 站点名称 |
| station_code | string | 站点编码 |
| current_count | int64 | 当前订单数量 |
| capacity | int | 站点容量 |
| usage_rate | float64 | 使用率（小数形式，如0.8） |
| usage_percent | string | 使用率（百分比字符串，如"80.0%"） |
| warning_level | string | 预警级别：normal/warning/critical |
| warning_message | string | 预警消息 |
| last_check_time | int64 | 最后检查时间（Unix时间戳） |
| recommend_action | string | 建议操作 |

## 预警消息和建议操作

### 正常状态（normal）
- **预警消息**: "库存正常，当前使用率X.X%"
- **建议操作**: "继续保持正常运营"

### 警告状态（warning）
- **预警消息**: "库存使用率较高，当前X.X%，建议关注"
- **建议操作**: "准备清理库存，加快货物流转速度"

### 严重状态（critical）
- **预警消息**: "库存严重超载！当前使用率X.X%，已达到严重预警线"
- **建议操作**: "立即清理库存，转移部分货物到其他站点"

## 使用场景

### 1. 实时监控
- 管理员定期查看所有站点的预警状态
- 重点关注警告和严重预警的站点
- 根据预警信息调整货物分配策略

### 2. 预防性管理
- 在达到严重预警前采取措施
- 提前安排货物转移或加快处理速度
- 避免因库存爆满影响正常运营

### 3. 决策支持
- 为站点扩容提供数据依据
- 优化站点间的货物分配
- 制定更合理的容量规划

## 业务流程

### 1. 预警检查
1. 系统查询所有启用的站点
2. 统计每个站点当前的订单数量
3. 计算容量使用率
4. 根据阈值判断预警级别
5. 生成预警消息和建议操作

### 2. 预警处理
1. 管理员查看预警信息
2. 分析预警原因和影响
3. 制定处理方案
4. 执行相应措施（转移货物、加快处理等）
5. 跟踪处理效果

## 配置说明

### 默认预警阈值
- **警告阈值**: 70%（可在代码中调整）
- **严重阈值**: 90%（可在代码中调整）

### 自定义阈值
未来可以考虑为每个站点设置不同的预警阈值，以适应不同站点的特点。

## 性能考虑

1. **实时计算**: 当前采用实时计算方式，适合中小规模站点
2. **缓存优化**: 对于大规模部署，建议添加Redis缓存
3. **定时更新**: 可以考虑定时计算预警数据，减少实时查询压力

## 扩展功能

- 短信通知：严重预警时发送短信

- 分析预警趋势和规律
- 生成预警统计报表

### 3. 智能建议
- 基于历史数据提供更精准的建议
- 自动推荐最优的货物转移方案
- 预测未来的库存压力
