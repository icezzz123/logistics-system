# 站点库存查询功能说明

## 功能概述

站点库存查询功能用于统计和查询各站点当前的订单库存情况，包括订单数量、状态分布、容量使用率和预警级别等信息。

## 业务说明

在物流系统中，"库存"指的是当前在某个站点的订单数量。通过查询订单的`current_station`字段和`status`字段，可以统计出每个站点的实时库存情况。

## API接口

### 查询站点库存

**接口地址**: `GET /api/stations/inventory`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| station_id | uint | 否 | 站点ID，不传或传0表示查询所有启用的站点 |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "station_id": 5,
        "station_name": "广州目的站",
        "station_code": "GZ-DEST-001",
        "total_orders": 4,
        "in_warehouse": 4,
        "sorting": 0,
        "in_transit": 0,
        "customs_clearing": 0,
        "dest_sorting": 0,
        "delivering": 0,
        "capacity_usage": "0.1%",
        "warning_level": "normal"
      }
    ]
  }
}
```

## 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| station_id | uint | 站点ID |
| station_name | string | 站点名称 |
| station_code | string | 站点编码 |
| total_orders | int64 | 当前站点总订单数 |
| in_warehouse | int64 | 已入库状态的订单数 |
| sorting | int64 | 分拣中状态的订单数 |
| in_transit | int64 | 运输中状态的订单数 |
| customs_clearing | int64 | 清关中状态的订单数 |
| dest_sorting | int64 | 目的地分拣状态的订单数 |
| delivering | int64 | 配送中状态的订单数 |
| capacity_usage | string | 容量使用率（百分比） |
| warning_level | string | 预警级别：normal/warning/critical |

## 预警级别说明

系统根据容量使用率自动计算预警级别：

- **normal（正常）**: 使用率 < 70%
- **warning（警告）**: 70% ≤ 使用率 < 90%
- **critical（严重）**: 使用率 ≥ 90%

## 统计逻辑

1. 查询所有启用的站点（status = 1）
2. 对每个站点，统计`current_station = station_id`的订单
3. 按订单状态分类统计数量
4. 计算容量使用率 = 总订单数 / 站点容量 × 100%
5. 根据使用率设置预警级别

## 使用场景

1. **站点管理员**: 查看本站点的实时库存情况
2. **调度员**: 监控各站点的库存分布，优化调度
3. **系统管理员**: 全局监控所有站点的库存状态
4. **容量预警**: 及时发现容量不足的站点

### 测试1: 查询所有站点库存

**请求**:
```
GET /api/stations/inventory
```

**预期结果**:
- 返回所有启用站点的库存信息
- 每个站点包含完整的统计数据
- 容量使用率正确计算

### 测试2: 查询指定站点库存

**请求**:
```
GET /api/stations/inventory?station_id=5
```

**预期结果**:
- 只返回指定站点的库存信息
- 统计数据准确

### 测试3: 容量预警测试

**场景**: 站点订单数接近或超过容量

**预期结果**:
- 使用率 < 70%: warning_level = "normal"
- 使用率 70-90%: warning_level = "warning"
- 使用率 ≥ 90%: warning_level = "critical"

## 实现细节

### 数据库查询优化

1. 使用索引字段查询（current_station、status）
2. 批量统计，减少数据库查询次数
3. 只查询启用的站点

### 性能考虑

- 对于大量站点，建议添加缓存
- 统计数据可以定期更新到Redis
- 支持按站点ID查询，减少不必要的计算
