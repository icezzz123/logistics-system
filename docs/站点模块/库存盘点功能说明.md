# 库存盘点功能说明

## 功能概述

库存盘点功能用于对站点当前的订单库存进行核对，生成盘点记录，发现库存差异，确保系统数据与实际情况一致。

## 业务说明

库存盘点是物流管理的重要环节，通过定期盘点可以：
- 发现系统数据与实际库存的差异
- 及时发现丢失、损坏的货物
- 提高库存管理的准确性
- 为异常处理提供数据支持

## 盘点类型

- **全盘（full）**: 对站点所有订单进行盘点
- **抽盘（spot）**: 对站点部分订单进行抽查盘点

## API接口

### 1. 创建库存盘点

**接口地址**: `POST /api/stations/inventory/check`

**请求参数**:
```json
{
  "station_id": 5,
  "check_type": "full",
  "remark": "月度全盘"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| station_id | uint | 是 | 站点ID |
| check_type | string | 是 | 盘点类型：full=全盘，spot=抽盘 |
| remark | string | 否 | 备注 |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "check_no": "CHK17731574073600",
    "message": "盘点单创建成功"
  }
}
```

### 3. 完成库存盘点

**接口地址**: `PUT /api/stations/inventory/check/{id}/complete`

**请求参数**:
```json
{
  "actual_count": 4,
  "remark": "盘点无差异"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| actual_count | int | 是 | 实际数量 |
| remark | string | 否 | 备注 |

### 4. 获取盘点列表

**接口地址**: `GET /api/stations/inventory/check`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| station_id | uint | 否 | 站点ID |
| check_type | string | 否 | 盘点类型 |
| status | int | 否 | 状态：1=盘点中，2=已完成 |
| start_time | int64 | 否 | 开始时间 |
| end_time | int64 | 否 | 结束时间 |
| page | int | 是 | 页码 |

## 业务流程

### 1. 创建盘点单

1. 选择要盘点的站点
2. 选择盘点类型（全盘/抽盘）
3. 填写备注信息
4. 系统自动生成盘点单号
5. 系统查询该站点当前所有订单
6. 创建盘点明细记录

### 2. 执行盘点

1. 工作人员根据盘点单进行实地盘点
2. 核对每个订单是否存在
3. 记录实际找到的订单数量

### 3. 完成盘点

1. 输入实际盘点数量
2. 系统自动计算差异
3. 填写盘点结果备注
4. 更新盘点状态为"已完成"

## 差异处理

### 差异类型

- **正差异（difference_count > 0）**: 实际数量 > 系统数量，可能有未录入系统的货物
- **负差异（difference_count < 0）**: 实际数量 < 系统数量，可能有货物丢失或损坏
- **无差异（difference_count = 0）**: 实际数量 = 系统数量，库存准确

### 差异处理建议

1. **正差异**: 检查是否有未扫描入库的货物，补录系统数据
2. **负差异**: 检查货物去向，可能需要创建异常记录
3. **重大差异**: 建议重新盘点确认

## 使用场景

1. **定期盘点**: 每月/季度进行全盘
2. **抽查盘点**: 不定期进行抽盘检查
3. **异常调查**: 发现问题时进行专项盘点
4. **交接盘点**: 人员交接时进行盘点

## 权限要求

- 创建盘点：站点管理员及以上权限
- 查看盘点：所有认证用户
- 完成盘点：盘点创建人或站点管理员

## 注意事项

1. 盘点期间建议暂停该站点的出入库操作
2. 盘点完成后及时处理发现的差异
3. 重要差异需要上报并创建异常记录
4. 定期分析盘点结果，优化库存管理流程
