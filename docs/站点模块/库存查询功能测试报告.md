### 发现的问题

1. **站点容量为0的处理**: 当前返回"N/A"，建议在创建站点时强制要求容量 > 0
2. **缓存优化**: 对于大量站点，建议添加Redis缓存

### 优化建议

1. **缓存策略**: 
   - 库存数据可以缓存5-10分钟
   - 使用Redis存储统计结果
   - 订单状态变更时清除相关缓存

2. **实时性优化**:
   - 考虑使用消息队列异步更新统计数据
   - 提供手动刷新接口

3. **扩展功能**:
   - 添加历史趋势统计
   - 支持按时间范围查询
   - 添加库存预警通知
