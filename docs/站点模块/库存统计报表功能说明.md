# 库存统计报表功能说明

## 功能概述

库存统计报表功能提供全面的库存数据分析和可视化支持，包括趋势分析、站点排行、状态分布等多维度统计，帮助管理人员深入了解库存状况，制定优化策略。

## 业务说明

库存统计报表是物流管理的重要决策支持工具，通过多维度数据分析，可以：
- 分析库存变化趋势，预测未来需求
- 识别高负载站点，优化资源配置
- 了解订单状态分布，发现业务瓶颈
- 评估站点容量利用率，指导扩容决策
- 为运营优化提供数据支撑

## API接口

### 1. 获取库存统计报表

**接口地址**: `GET /api/stations/inventory/stats`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| station_id | uint | 否 | 站点ID，0或不传表示所有站点 |
| date_type | string | 否 | 统计类型：day=日统计，week=周统计，month=月统计，默认day |
| start_date | string | 否 | 开始日期 YYYY-MM-DD，默认7天前 |
| end_date | string | 否 | 结束日期 YYYY-MM-DD，默认今天 |

## 响应字段说明

### 统计汇总字段 (summary)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| total_stations | int | 总站点数 |
| total_orders | int64 | 总订单数 |
| avg_usage_rate | float64 | 平均使用率 |
| max_usage_rate | float64 | 最高使用率 |
| min_usage_rate | float64 | 最低使用率 |
| warning_stations | int | 警告状态站点数 |
| critical_stations | int | 严重预警站点数 |

### 趋势数据字段 (trends)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| date | string | 日期 |
| total_orders | int64 | 当日总订单数 |
| avg_usage | float64 | 当日平均使用率 |
| max_usage | float64 | 当日最高使用率 |
| min_usage | float64 | 当日最低使用率 |
| in_warehouse | int64 | 已入库数量 |
| sorting | int64 | 分拣中数量 |
| in_transit | int64 | 运输中数量 |
| delivering | int64 | 配送中数量 |

### 站点排行字段 (station_top)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| station_id | uint | 站点ID |
| station_name | string | 站点名称 |
| station_code | string | 站点编码 |
| total_orders | int64 | 总订单数 |
| capacity | int | 站点容量 |
| usage_rate | float64 | 使用率（小数） |
| usage_percent | string | 使用率（百分比） |
| warning_level | string | 预警级别 |
| in_warehouse | int64 | 已入库数量 |
| sorting | int64 | 分拣中数量 |
| in_transit | int64 | 运输中数量 |
| delivering | int64 | 配送中数量 |

### 状态分布字段 (status_distribution)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| status | int | 状态值 |
| status_name | string | 状态名称 |
| count | int64 | 数量 |
| percentage | string | 占比 |

### 站点分布字段 (station_distribution)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| station_id | uint | 站点ID |
| station_name | string | 站点名称 |

| 字段名 | 类型 | 说明 |

| percentage | string | 占比 |

## 统计维度

### 1. 时间趋势分析
- **日统计**: 按天统计库存变化
- **周统计**: 按周统计库存趋势
- **月统计**: 按月统计库存规律

### 2. 站点排行分析
- 按使用率排序，识别高负载站点
- 显示各站点的详细库存构成
- 提供预警级别判断

### 3. 状态分布分析
- 统计各订单状态的数量和占比
- 识别业务流程中的瓶颈环节
- 帮助优化作业流程

### 4. 容量分布分析
- 统计不同使用率区间的站点数量
- 评估整体容量利用情况
- 指导容量规划和扩容决策

## 使用场景

### 1. 日常监控
- 管理员查看整体库存状况
- 监控各站点的负载情况
- 及时发现异常和瓶颈

### 2. 趋势分析
- 分析库存变化规律
- 预测未来库存需求
- 制定资源配置策略

### 3. 性能优化
- 识别高负载站点
- 优化货物分配策略
- 提高整体运营效率

### 4. 决策支持
- 为扩容决策提供数据支撑
- 评估运营策略效果
- 制定长期发展规划

## 数据计算逻辑

### 1. 趋势数据计算
- 按指定日期范围和统计类型生成时间序列
- 对每个时间点统计各站点的库存情况
- 计算平均值、最大值、最小值等统计指标

### 2. 站点排行计算
- 统计每个站点当前的订单数量
- 按订单状态分类统计
- 计算使用率并确定预警级别
- 按使用率降序排列

### 3. 分布统计计算
- **状态分布**: 统计所有订单按状态的分布情况
- **站点分布**: 统计订单在各站点的分布情况
- **容量分布**: 统计站点按使用率区间的分布情况

## 性能优化

### 1. 查询优化
- 使用数据库索引加速查询
- 批量查询减少数据库访问次数
- 合理使用聚合函数

### 2. 缓存策略
- 对于大规模部署，建议添加Redis缓存
- 缓存热点数据，减少重复计算
- 设置合理的缓存过期时间

### 3. 分页处理
- 对于大量数据，支持分页查询
- 限制单次查询的数据量
- 提供数据导出功能
