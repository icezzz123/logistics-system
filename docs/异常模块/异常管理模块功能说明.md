# 异常管理模块功能说明

## 功能概述
异常管理模块用于处理物流作业过程中的异常件，包括异常创建、处理人分配、异常处理、异常关闭以及统计分析。模块与订单状态机联动，创建异常时可将订单推进为“异常”，关闭时支持恢复订单状态。

---

## 已实现功能

### 1. 异常创建
- 支持按订单创建异常记录
- 支持选择异常类型、站点、异常描述、图片列表、备注
- 创建异常时自动生成异常单号
- 若订单当前允许进入异常状态，会同步将订单状态更新为 `异常`
- 限制同一订单只能存在一条未关闭异常记录

### 2. 异常分配
- 支持为异常单分配处理人
- 处理人角色校验为站点管理员、调度员或管理员
- 已关闭异常不可重复分配

### 3. 异常处理
- 支持将异常更新为“处理中”或“已解决”
- 支持填写处理方案、处理结果、赔付金额、备注
- 已解决时要求至少填写方案或结果之一

### 4. 异常关闭
- 仅“已解决”的异常允许关闭
- 支持补充关闭备注和结果
- 支持可选恢复订单状态，例如从 `异常` 恢复到 `运输中`
- 恢复订单状态时遵循现有订单状态机规则

### 5. 异常查询
- 支持异常列表分页查询
- 支持按订单号、异常类型、状态、站点、上报人、处理人、上报时间范围筛选
- 支持单条异常详情查询

### 6. 异常统计分析
- 支持异常总数、待处理、处理中、已解决、已关闭数量统计
- 支持赔付金额汇总
- 支持按异常类型、状态、站点、日期统计

---

## API接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/exceptions | 创建异常 |
| GET | /api/exceptions | 获取异常列表 |
| GET | /api/exceptions/:id | 获取异常详情 |
| PUT | /api/exceptions/:id/assign | 分配处理人 |
| PUT | /api/exceptions/:id/process | 处理异常 |
| PUT | /api/exceptions/:id/close | 关闭异常 |
| GET | /api/exceptions/stats | 获取异常统计 |

---

## 权限说明
- 查询类接口要求具备 `exception:view`
- 创建接口要求具备 `exception:create`
- 分配、处理、关闭要求具备 `exception:update`
- 当前权限设计下，站点管理员、调度员、管理员具备异常处理能力

---

## 关键业务规则
- 已关闭异常不可再次分配或处理
- 未分配处理人的异常不能直接进入处理流程
- 同一订单存在未关闭异常时，不允许重复创建新异常
- 关闭异常时，如果指定 `resume_status`，则订单必须当前处于 `异常` 状态
- 订单恢复状态时遵循订单状态机权限和转换规则

## 请求示例

### 创建异常
```json
{
  "order_id": 31,
  "type": 3,
  "station_id": 1,
  "description": "运输节点长时间停滞，疑似延误",
  "images": [
    "https://example.com/img1.jpg"
  ],
  "remark": "客服反馈异常"
}
```

### 分配处理人
```json
{
  "handler_id": 5,
  "remark": "分配给站点管理员处理"
}
```

### 处理异常
```json
{
  "status": 3,
  "solution": "重新安排运输车辆",
  "result": "问题已解决，等待恢复订单流程",
  "compensate_amount": 30,
  "remark": "已完成处理"
}
```

### 关闭异常
```json
{
  "resume_status": 5,
  "remark": "关闭异常并恢复订单运输流程",
  "result": "异常关闭"
}
```
