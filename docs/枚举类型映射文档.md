# 枚举类型映射文档

本系统所有枚举类型在数据库中使用整数存储，从1开始编号。

## 1. 用户角色 (UserRole)

| 枚举值 | 数据库值 | 说明 |
|--------|---------|------|
| RoleCustomer | 1 | 客户（默认） |
| RoleCourier | 2 | 快递员 |
| RoleSorter | 3 | 分拣员 |
| RoleDriver | 4 | 司机 |
| RoleSiteManager | 5 | 站点管理员 |
| RoleDispatcher | 6 | 调度员 |
| RoleAdmin | 7 | 管理员 |

**数据库字段：** `role INT NOT NULL DEFAULT 1`

**角色权限说明：**
- **客户(1)**: 下单、查询自己的订单、追踪货物
- **快递员(2)**: 揽件、派件、签收确认
- **分拣员(3)**: 扫描分拣、记录分拣信息
- **司机(4)**: 运输任务执行、货物装卸
- **站点管理员(5)**: 管理站点、货物出入库、异常处理
- **调度员(6)**: 任务分配、路线规划、资源调度
- **管理员(7)**: 系统管理、用户管理、全局配置

## 使用示例

### Go 代码中使用

```go
// 创建订单时设置状态
order := models.Order{
    Status: models.OrderPending, // 值为 1
    TransportMode: models.TransportAir, // 值为 1
}

// 创建用户时设置角色
user := models.User{
    Role: models.RoleCustomer, // 值为 5（默认）
}

// 查询特定状态的订单
db.Where("status = ?", models.OrderInTransit).Find(&orders)

// 查询特定角色的用户
db.Where("role = ?", models.RoleDriver).Find(&users)
```

### SQL 查询示例

```sql
-- 查询待处理的订单
SELECT * FROM orders WHERE status = 1;

-- 查询所有管理员
SELECT * FROM users WHERE role = 7;

-- 查询所有快递员
SELECT * FROM users WHERE role = 2;

-- 查询空运订单
SELECT * FROM orders WHERE transport_mode = 1;

-- 查询已签收的订单数量
SELECT COUNT(*) FROM orders WHERE status = 10;

-- 查询异常订单（破损类型）
SELECT * FROM exception_records WHERE type = 1;

-- 统计各状态订单数量
SELECT status, COUNT(*) as count 
FROM orders 
GROUP BY status;

-- 统计各角色用户数量
SELECT role, COUNT(*) as count 
FROM users 
GROUP BY role;
```

## 注意事项

1. **默认值**：
   - 用户角色默认为 1 (客户)
   - 订单状态默认为 1 (待处理)
   - 异常状态默认为 1 (待处理)

2. **非空约束**：
   - 所有枚举字段都设置了 NOT NULL 约束
   - 必须提供有效的枚举值

3. **索引优化**：
   - 高频查询的枚举字段（如 status, role）已建立索引
   - 整数类型索引效率高于字符串

4. **扩展性**：
   - 新增枚举值时，使用下一个连续整数
   - 不要修改已有枚举值的数字映射
   - 删除枚举值时要谨慎，确保数据库中没有使用该值的记录

5. **类型转换**：
   ```go
   // 整数转枚举
   status := models.OrderStatus(1) // OrderPending
   
   // 枚举转整数
   statusInt := int(models.OrderPending) // 1
   ```

6. **前后端协议**：
   - API 返回和接收的都是整数值
   - 前端负责将整数映射为显示文本
   - 不要在 API 中传输字符串形式的枚举

## 数据迁移注意事项

如果从字符串枚举迁移到整数枚举：

```sql
-- 示例：迁移订单状态
UPDATE orders SET status = 1 WHERE status = 'pending';
UPDATE orders SET status = 2 WHERE status = 'accepted';
UPDATE orders SET status = 3 WHERE status = 'in_warehouse';
-- ... 其他状态

-- 修改字段类型
ALTER TABLE orders MODIFY COLUMN status INT NOT NULL DEFAULT 1;
```
