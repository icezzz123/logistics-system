### 4.1 设计目标与原则（约600字，1页）

#### 4.1.1 系统设计目标（约250字）
**写作要点**:
- 围绕跨境物流全流程管理展开设计
- 满足订单、站点、分拣、运输、追踪、异常、调度等业务落地
- 支撑多角色协同作业
- 为后续实现、测试和部署提供结构化基础

**配图安排**:
- 原则上不单独配图
- 说明：本节以文字交代设计导向即可，重点在于承接第三章需求分析结果

#### 4.1.2 系统设计原则（约350字）
**写作要点**:
- 模块化与分层设计
- 高内聚、低耦合
- 可维护性与可扩展性
- 安全性与可靠性
- 面向实际业务场景的可操作性

**配图安排**:

### 4.2 系统总体架构设计（约1300字，2页）

#### 4.2.1 总体技术架构设计（约500字）
**写作要点**:
- 前后端分离架构
- 前端采用 Vue 3 + Vite + Element Plus + Pinia
- 后端采用 Go + Gin + GORM
- 数据层采用 MySQL，Redis作为可选缓存
- JWT承担认证，RBAC承担权限控制

**配图安排**:
- **图4-1：系统总体技术架构图**（必配）
- **插入位置**：本节正文中后部
- **图示内容**：展示“前端界面层 → 接口访问层 → 后端业务层 → 数据存储层”的总体结构，并标注 Vue、Gin、MySQL、Redis、JWT 等关键技术
- **写作配合方式**：正文先概述技术路线，再结合架构图说明各层职责和数据流向

#### 4.2.2 分层架构设计（约450字）
**写作要点**:
- 中间件层、控制器层、服务层、模型层、数据库层的划分
- 各层职责边界
- 参数绑定、业务校验、事务控制、数据持久化的分工
- 分层设计对维护和扩展的价值

**配图安排**:
- **图4-2：后端分层架构图**（必配）
- **插入位置**：本节正文之后
- **图示内容**：展示“Middleware → Controller → Service → Model → Database”的调用链路
- **写作配合方式**：正文解释分层思想，图用于直观呈现后端代码组织结构

#### 4.2.3 系统运行机制设计（约350字）
**写作要点**:
- 请求从路由进入中间件，再进入控制器和服务层的处理流程
- JWT认证、日志记录、异常恢复在请求链中的位置
- 统一响应封装机制
- 事务与状态日志在关键业务中的作用

**配图安排**:
- 原则上不单独新增图表

## 写作注意事项

### 风格要求
- 这一章重点不是“系统需要什么”，而是“系统准备怎样设计”
- 尽量多写结构、流程、表、字段、层次关系，少写泛泛而谈的功能介绍
- 描述要紧贴当前项目真实实现，尤其是 Go + Gin + GORM + MySQL + Redis、JWT、RBAC、统一扫码、状态日志、事务控制这些内容
- 第四章可以自然使用“如图4-1所示”“表4-1列出了”等表达，让章节更像正式论文

### 结构要求
- 4.1 先交代设计目标和原则
- 4.2 解决“系统整体怎么搭起来”
- 4.3 解决“各模块怎么设计”
- 4.4 解决“数据怎么落库”
- 4.5 解决“接口和关键技术怎么支撑”
- 4.6 用一小段完成收束
