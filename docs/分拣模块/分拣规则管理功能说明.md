# 分拣规则管理功能说明

## 功能概述

分拣规则管理功能是物流系统的核心组件，通过智能路由匹配算法，自动为订单分配最优的目标站点，实现高效的货物分拣和路由规划。

## 业务说明

分拣规则管理系统通过预设的地址规则，实现：
- 自动路由匹配：根据收件地址自动匹配最合适的分拣规则
- 优先级管理：支持规则优先级设置，确保精确匹配优先
- 智能算法：采用多级匹配算法，从精确到模糊逐级匹配
- 灵活配置：支持国家、省份、城市、区县多级地址配置
- 批量处理：支持批量地址路由匹配，提高处理效率

## 核心功能

### 1. 分拣规则CRUD管理
- 创建分拣规则
- 查询规则列表（支持分页、搜索、筛选）
- 更新规则信息
- 启用/禁用规则
- 删除规则（带安全检查）

### 2. 智能路由匹配算法
- **精确匹配**：国家+省份+城市+区县完全匹配
- **城市级匹配**：国家+省份+城市匹配（区县为空）
- **省份级匹配**：国家+省份匹配（城市、区县为空）
- **国家级匹配**：国家匹配（省份、城市、区县为空）

### 3. 优先级管理
- 支持数值优先级设置（数值越大优先级越高）
- 同优先级按创建时间排序
- 优先级分组统计和管理

## API接口

### 1. 创建分拣规则

**接口地址**: `POST /api/sorting/rules`

**请求参数**:
```json
{
  "rule_name": "中国广东省广州市规则",
  "country": "中国",
  "province": "广东省", 
  "city": "广州市",
  "district": "",
  "route_code": "CN-GD-GZ",
  "station_id": 5,
  "priority": 200,
  "description": "中国广东省广州市的分拣规则"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 2,
    "message": "分拣规则创建成功"

}
```

## 路由匹配算法

### 匹配优先级（从高到低）

1. **精确匹配（exact）**
   - 条件：国家+省份+城市+区县完全匹配
   - 适用：具体到区县的精确地址

2. **城市级匹配（city）**
   - 条件：国家+省份+城市匹配，规则中区县为空
   - 适用：城市级别的通用规则

3. **省份级匹配（province）**
   - 条件：国家+省份匹配，规则中城市、区县为空
   - 适用：省份级别的通用规则

4. **国家级匹配（country）**
   - 条件：国家匹配，规则中省份、城市、区县为空
   - 适用：国家级别的通用规则

### 优先级处理

- 在同一匹配级别内，按优先级数值降序排列
- 优先级相同时，按创建时间升序排列（先创建的优先）
- 返回第一个匹配的规则

### 建议规则

- 当找到匹配规则时，同时返回相似规则作为建议
- 建议规则基于同国家的其他规则
- 最多返回5个建议规则

## 业务规则

### 1. 规则冲突检查
- 不允许创建完全相同地址的规则
- 更新规则时检查地址冲突
- 只检查启用状态的规则

### 2. 站点验证
- 创建/更新规则时验证目标站点存在且启用
- 站点被禁用时相关规则仍可使用

### 3. 删除限制
- 已被分拣记录使用的规则不能删除
- 建议禁用而非删除已使用的规则

### 4. 优先级管理
- 支持负数、零、正数优先级
- 数值越大优先级越高
- 优先级分组统计便于管理
