# 分拣作业功能说明

## 功能概述

分拣作业功能是物流系统的核心模块之一，实现了完整的分拣作业流程管理，包括分拣任务管理、扫描作业、进度跟踪和统计分析等功能。

## 主要功能

### 1. 分拣任务管理

#### 1.1 任务创建与分配
- **任务创建**: 支持创建分拣任务，设置站点、分拣员、预计数量等
- **任务分配**: 支持将任务分配给指定的分拣员
- **任务编号**: 自动生成唯一的任务编号（格式：ST+时间戳）
- **预计数量**: 支持设置任务的预计分拣数量，用于进度计算

#### 1.2 任务状态管理
- **待处理**: 任务创建后的初始状态
- **处理中**: 任务开始执行，记录开始时间
- **已完成**: 任务完成，记录结束时间
- **已取消**: 任务取消

#### 1.3 任务进度跟踪
- **实时进度**: 根据扫描记录自动更新任务进度
- **进度百分比**: 已分拣数量/预计总数量 × 100%
- **耗时统计**: 记录任务开始和结束时间，计算耗时

### 2. 分拣扫描功能

#### 2.1 扫描录入
- **订单扫描**: 扫描订单进行分拣处理
- **自动路由匹配**: 根据收货地址自动匹配分拣规则
- **目标站点确定**: 根据匹配规则确定目标站点
- **扫描记录**: 自动创建分拣记录，记录扫描时间和操作员

#### 2.2 路由匹配算法
- **精确匹配**: 国家+省份+城市+区县完全匹配
- **城市级匹配**: 国家+省份+城市匹配
- **省份级匹配**: 国家+省份匹配
- **国家级匹配**: 国家匹配
- **建议规则**: 未匹配时提供相似规则建议

#### 2.3 扫描结果处理
- **成功分拣**: 匹配到规则，确定目标站点
- **异常处理**: 未匹配规则时提供建议，需要人工处理
- **重复检查**: 防止同一订单在同一站点重复扫描

### 3. 分拣记录管理

#### 3.1 记录查询
- **多条件筛选**: 支持按任务、订单、站点、分拣员等条件查询
- **时间范围**: 支持按扫描时间范围查询
- **正确性筛选**: 支持按分拣正确性查询
- **分页查询**: 支持大数据量的分页查询

#### 3.2 记录详情
- **基本信息**: 任务编号、订单号、站点信息
- **路由信息**: 匹配规则、路由代码、目标站点
- **操作信息**: 分拣员、扫描时间、正确性标记
- **关联信息**: 显示相关的任务、订单、站点名称

### 4. 统计分析功能

#### 4.1 任务统计
- **任务数量**: 各状态任务数量统计
- **进度统计**: 平均进度、总件数、已分拣件数
- **完成率**: 任务完成率统计

#### 4.2 分拣记录统计
- **总体统计**: 总记录数、正确记录数、错误记录数
- **准确率**: 分拣准确率计算
- **效率统计**: 平均扫描时间间隔

#### 4.3 分拣员统计
- **个人统计**: 每个分拣员的任务数、记录数、准确率
- **效率排行**: 按分拣数量和准确率排行
- **平均速度**: 计算分拣员的平均分拣速度

#### 4.4 站点统计
- **站点分拣量**: 各站点的分拣任务和记录统计
- **站点效率**: 站点分拣效率分析

## 业务规则

### 分拣任务规则
1. 任务编号自动生成，格式为ST+时间戳
2. 任务状态转换规则：
   - 待处理 → 处理中、已取消
   - 处理中 → 已完成、已取消
   - 已完成、已取消 → 不能转换
3. 分拣员必须具有相应权限（role >= 2）
4. 任务进度根据扫描记录自动更新

### 分拣扫描规则
1. 同一订单在同一站点不能重复扫描
2. 扫描时自动进行路由匹配
3. 匹配失败时提供建议规则
4. 扫描成功后自动更新任务进度

### 路由匹配规则
1. 按精确度从高到低进行匹配
2. 优先级高的规则优先匹配
3. 未匹配时提供相似规则建议
4. 支持多级地址匹配（国家→省份→城市→区县）

### 数据完整性
1. 扫描前验证订单和站点的有效性
2. 记录创建时关联相应的任务和规则
3. 统计数据实时计算，确保准确性

- **统计算法**: 多维度数据统计和分析

## 业务流程

### 标准分拣流程
1. **任务创建**: 管理员创建分拣任务，设置预计数量
2. **任务分配**: 将任务分配给分拣员
3. **开始作业**: 分拣员开始处理，任务状态变为"处理中"
4. **扫描分拣**: 逐个扫描订单，系统自动路由匹配
5. **进度跟踪**: 系统实时更新任务进度
6. **完成任务**: 分拣完成后，任务状态变为"已完成"

### 异常处理流程
1. **路由未匹配**: 提供建议规则，人工处理
2. **重复扫描**: 系统提示已扫描，防止重复操作
3. **任务取消**: 支持取消未完成的任务
