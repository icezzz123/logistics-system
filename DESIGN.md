# Logistics UI Design Guide

## 1. Design Direction

This project should not copy Apple literally. The right direction is:

- keep the existing logistics orange brand theme
- borrow Apple's restraint, spacing, and interaction polish
- avoid turning the system into a generic black-and-white tech landing page

In short: **orange business identity + Apple-style interaction discipline**.

The system is still a cross-border logistics operation platform. It should feel warm, trustworthy, and operational. The interface should be cleaner and more refined than a common admin panel, but it still needs to communicate status, flow, traceability, and role responsibility.

## 2. Visual Language

### Brand colors

- Primary orange: `#ee4d2d`
- Deep orange: `#cf3719`
- Soft orange: `#ffd9cc`

These remain the core brand colors. Do not replace them with Apple's blue as the main identity color.

### Neutral surfaces

- Main background: `#fff7f2`
- Panel surface: `#fffaf6`
- Strong surface: `#ffffff`
- Text primary: `#2e221d`
- Text secondary: `#74645b`

### Interaction accent

For focus states and polished controls, a cleaner Apple-like interaction rhythm is allowed, but the visible action color should still stay inside the orange system unless there is a strong reason to use blue for accessibility.

## 3. What to Borrow from Apple

Borrow these qualities:

- cleaner hierarchy
- stronger whitespace control
- calmer component surfaces
- rounded, pill-like buttons where appropriate
- more restrained shadows
- more deliberate visual emphasis

Do not borrow these blindly:

- full black/white section rhythm as the default
- pure Apple product-marketing layout
- blue replacing orange as the main brand color

## 4. Login Page Guidance

The login page may use an Apple-inspired split:

- left side: strong brand canvas
- right side: quiet form area

But the left side should still use the logistics orange brand atmosphere, not pure black as the permanent default.

The card on the right should feel:

- calm
- bright
- well-spaced
- highly readable

Buttons should feel more Apple-like:

- simpler
- more precise
- fuller radius
- cleaner shadows

Yet their color should remain orange-led.

## 5. Component Rules

### Buttons

- Primary buttons should use orange, not blue
- Buttons may use fuller radius, including pill shapes where appropriate
- Avoid overly heavy gradients; prefer cleaner color blocks or soft orange depth
- Hover and active states should be subtle and controlled

### Inputs

- Inputs should feel lighter and cleaner than traditional admin forms
- Use soft radius and understated borders
- Focus states should be clear, but not noisy

### Cards

- Keep white or warm off-white cards
- Use lighter, more refined shadows than before
- Borders should remain subtle

## 6. Typography

- Keep strong headline presence
- Use cleaner, more controlled heading rhythm
- Avoid decorative excess
- Operational pages should favor clarity over drama

The login page can be slightly more polished and brand-led, but business pages should still feel efficient first.

## 7. Practical Rule for This Project

When designing new UI:

1. Keep the orange logistics identity
2. Use Apple-style restraint for buttons, spacing, and surfaces
3. Make business actions obvious
4. Do not sacrifice usability for visual purity

## 8. Do

- keep orange as the visible brand center
- simplify button styling
- use fuller rounded corners on key CTAs
- use cleaner card and form treatment
- preserve the existing logistics product character

## 9. Do Not

- do not turn the whole system into pure black-and-white Apple imitation
- do not replace orange CTAs with blue CTAs by default
- do not over-style business pages
- do not remove operational density where users need it
