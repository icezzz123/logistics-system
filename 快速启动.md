# 快速启动指南

## 前置条件

确保已安装以下软件：
- Go 1.21 或更高版本
- MySQL 8.0 或更高版本
- Redis 6.0 或更高版本

## 步骤1：克隆项目

```bash
git clone <repository-url>
cd logistics-system
```

## 步骤2：安装依赖

```bash
go mod download
# 或使用 make 命令
make deps
```

## 步骤3：配置环境变量

1. 复制环境变量示例文件：
```bash
cp .env.example .env
```

2. 编辑 `.env` 文件，配置数据库和Redis连接信息：

```env
# 服务器配置
SERVER_PORT=8080
SERVER_MODE=debug

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=logistics_system
DB_CHARSET=utf8mb4

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# JWT配置
JWT_SECRET=your_jwt_secret_key_change_this_in_production
JWT_EXPIRE_HOURS=24
```

## 步骤4：创建数据库

连接到MySQL并执行：

```sql
CREATE DATABASE logistics_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 步骤5：启动Redis

确保Redis服务正在运行：

```bash
# Windows
redis-server

# Linux/Mac
sudo systemctl start redis
# 或
redis-server
```

## 步骤6：运行项目

### 方式1：直接运行（推荐开发时使用）

```bash
go run main.go
# 或使用 make 命令
make run
```

### 方式2：编译后运行

```bash
# 编译
go build -o bin/logistics-system.exe main.go
# 或使用 make 命令
make build

# 运行
./bin/logistics-system.exe
```

## 步骤7：验证服务

打开浏览器或使用curl访问：

```bash
curl http://localhost:8080/health
```

应该返回：
```json
{
  "status": "ok",
  "message": "Logistics System API is running"
}
```

## 常用API测试

### 1. 用户注册

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "123456",
    "email": "test@example.com",
    "phone": "13800138000",
    "real_name": "测试用户"
  }'
```

### 2. 用户登录

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "123456"
  }'
```

返回的token用于后续需要认证的请求。

### 3. 获取用户信息（需要token）

```bash
curl -X GET http://localhost:8080/api/user/info \
  -H "Authorization: Bearer <your_token>"
```

### 4. 创建订单（需要token）

```bash
curl -X POST http://localhost:8080/api/orders \
  -H "Content-Type: application/json" \
```

## 常见问题

### 1. 数据库连接失败

检查：
- MySQL服务是否启动
- `.env` 文件中的数据库配置是否正确
- 数据库是否已创建

### 2. Redis连接失败

检查：
- Redis服务是否启动
- `.env` 文件中的Redis配置是否正确

### 3. 端口被占用

修改 `.env` 文件中的 `SERVER_PORT` 为其他端口。

### 4. JWT token无效

检查：
- token是否已过期
- 请求头中是否正确设置了 `Authorization: Bearer <token>`
