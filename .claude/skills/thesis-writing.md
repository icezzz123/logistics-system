# 论文写作优化 SKILL

## 核心原则

在处理论文相关内容时，必须遵循以下原则，使文本更自然、更像人类书写：

### 0. 【硬性要求】禁止虚构任何内容 + 不主动生成文档
- **绝对禁止**: 
  - 虚构研究者姓名（如"张明"、"Smith"等）
  - 虚构文献引用和研究内容
  - 虚构数据、统计结果、实验结果
  - 虚构企业案例、技术方案
  - 编造任何未经证实的事实
  
- **必须做到**:
  - 只使用用户提供的真实信息
  - 只描述用户实际实现的系统功能
  - 如需引用文献，明确告知用户需要自行查阅和补充
  - 如需数据支撑，明确标注"需补充真实数据"
  - 涉及研究现状时，提供框架和思路，由用户填充真实文献
  
- **违反后果**: 虚构内容会导致学术不端，严重影响论文质量和学生信誉

- **【硬性要求】不主动生成文档**:
  - **默认行为**: 直接在对话中输出文本内容，不使用 Write 工具创建文档
  - **例外情况**: 仅当用户明确要求"生成文档"、"保存到文件"、"创建文档"时才使用 Write 工具
  - **原因**: 避免产生不必要的文件，用户可以直接复制对话中的内容到自己的文档
  - **适用范围**: 论文写作、修改、润色、翻译等所有论文相关任务

**示例对比**:
```
❌ 虚构内容：张明等人研究了跨境物流信息平台的架构设计，提出了基于SOA的分布式系统架构...

✅ 正确做法：
方案1：[此处需补充真实文献] 研究了跨境物流信息平台的架构设计...
方案2：在物流信息平台架构方面，国内外学者进行了大量研究，主要集中在分布式架构、微服务架构等方向。[具体文献需用户自行查阅补充]
```

### 1. 长短句结合
- **避免**: 连续使用长句或短句
- **要求**: 
  - 长句和短句交替使用，形成节奏感
  - 长句用于阐述复杂逻辑，短句用于强调重点
  - 一段中至少包含 2-3 种不同长度的句子

**示例对比**:
```
❌ 机械化：系统采用前后端分离架构。后端基于Go语言开发。前端采用Vue 3构建。系统实现了多个模块。

✅ 自然化：系统采用前后端分离架构，后端基于Go语言开发，前端采用Vue 3构建。通过这种架构，系统实现了订单管理、物流追踪等多个核心模块。
```

### 2. 避免过于机械化的段落结构
- **避免**: 
  - "首先...其次...再次...最后"的刻板结构
  - "一方面...另一方面"的对称结构
  - 每句话都以"系统"、"本文"开头
  - 重复使用"在...方面"、"在...上"等相同的过渡短语
  
- **要求**:
  - 使用自然的过渡和衔接
  - 适当省略过渡词，让逻辑自然流动
  - 变化句子的主语和开头方式
  - 多样化过渡表达，避免重复

**示例对比**:
```
❌ 机械化：首先，系统实现了用户管理功能。其次，系统实现了订单管理功能。再次，系统实现了物流追踪功能。最后，系统实现了异常管理功能。

✅ 自然化：系统实现了用户管理、订单管理等核心功能。在物流追踪方面，通过实时记录订单状态变化，用户可以随时查询货物位置。异常管理模块则确保了问题能够及时发现和处理。
```

### 3. 适当的不规则表达 + 口语化学术风格
- **避免**: 过于完美、过于工整的句式
- **要求**:
  - 允许适度的口语化表达（在学术规范内）
  - 使用一些不那么"标准"但更自然的表达方式
  - 避免每个句子都是完美的主谓宾结构
  
- **推荐的口语化表达**（保持学术性的前提下）:
  - "这块" / "这方面" / "这个方向"
  - "挺多" / "不少" / "比较多"
  - "还不错" / "效果不错" / "比较好"
  - "其实" / "确实" / "的确"
  - "比较" / "相对" / "稍微"
  - "话说回来" / "不过" / "但是"
  - "慢慢成了" / "逐渐成为"
  - "能有效" / "可以较好地"
  - "问题就来了" / "问题出现了"
  - "还是挺...的" / "确实比较..."

**示例对比**:
```
❌ 过于正式：本系统在业务流程完整性、角色协同效率和物流信息可追溯性三个方面均具有较好的实现效果。

✅ 口语化学术：系统在业务流程完整性和角色协同效率方面表现还不错，物流信息的可追溯性也得到了比较好的实现。

❌ 过于正式：现有研究主要集中于物流追踪技术、供应链优化和智能化应用等领域。

✅ 口语化学术：现有研究主要集中在物流追踪技术、供应链优化和智能化应用这几个方向。

❌ 过于正式：该研究为物流系统中多角色协同作业的权限控制提供了理论参考。

✅ 口语化学术：这个研究为物流系统中多角色协同作业的权限控制提供了不错的参考。
```

**自然过渡词的使用**:
```
✅ 推荐使用（多样化）：
- 直接陈述：系统实现了...、本文设计了...
- 针对性过渡：针对...问题、围绕...需求、基于...特点
- 功能性过渡：通过...、采用...、利用...
- 结果性过渡：从而...、进而...、使得...
- 补充性过渡：此外、另外、同时、与此同时
- 对比性过渡：相比之下、不同于...、与...不同

❌ 避免重复使用：
- 连续使用"在...方面"（如：在设计方面、在实现方面、在测试方面）
- 连续使用"在...上"（如：在功能上、在性能上、在安全上）
- 连续使用"首先...其次...再次...最后"
- 连续使用"一方面...另一方面"

✅ 替代方案示例：
❌ 在系统设计方面，采用了前后端分离架构。在权限管理方面，使用了RBAC模型。在数据存储方面，选择了MySQL数据库。

✅ 系统采用前后端分离架构。权限管理使用RBAC模型。数据存储选择MySQL数据库。

✅ 系统采用前后端分离架构。针对权限管理需求，引入了RBAC模型。数据存储层则选择了MySQL数据库。
```

### 4. 保留原文核心意思和意图
- **要求**:
  - 不改变原文的核心观点
  - 不删除关键信息
  - 不添加原文没有的内容
  - 保持原文的逻辑结构和论证思路

### 5. 句式多样化
- **要求**:
  - 主动句和被动句结合
  - 陈述句、因果句、条件句混合使用
  - 避免连续使用相同的句式结构

**示例对比**:
```
❌ 单一句式：系统实现了订单管理。系统实现了物流追踪。系统实现了异常处理。

✅ 多样句式：系统实现了订单管理功能。通过实时追踪，物流信息得以透明化。当出现异常时，系统能够及时处理并反馈。
```

### 6. 自然的连接和过渡
- **避免**: 生硬的连接词堆砌
- **要求**:
  - 使用隐性过渡（通过内容的逻辑关系自然衔接）
  - 适当使用连接词，但不过度
  - 让句子之间的关系通过语义自然呈现

### 7. 适度的细节和留白
- **避免**: 把所有细节都写得很满
- **要求**:
  - 重点内容详细阐述
  - 次要内容适当概括
  - 给读者留出理解和思考的空间

## 应用场景

### 场景 1: 摘要写作/优化
- 确保逻辑清晰，层次分明
- 长短句结合，避免单调
- 突出核心创新点
- 字数控制在要求范围内

### 场景 2: 正文写作/润色
- 段落之间自然过渡
- 避免机械化的"首先、其次、最后"
- 使用多样化的句式
- 保持学术规范的同时增加可读性

### 场景 3: 中英文翻译
- 英文翻译要符合英文表达习惯
- 避免中式英语
- 保持原文的核心意思
- 句式要自然流畅

### 场景 4: 关键词提取
- 提取核心概念
- 符合学术规范
- 体现研究重点

## 检查清单

在完成论文写作/优化后，检查以下项目：

- [ ] **【最重要】是否包含任何虚构内容？（人名/文献/数据/案例）**
- [ ] **【重要】是否主动生成了文档？（除非用户明确要求）**
- [ ] 是否有连续3个以上相似长度的句子？
- [ ] 是否使用了"首先...其次...最后"的机械结构？
- [ ] 是否每句话都以相同的主语开头？
- [ ] 句式是否有变化（主动/被动、陈述/因果等）？
- [ ] 过渡是否自然，还是依赖大量连接词？
- [ ] 是否保留了原文的核心意思和关键信息？
- [ ] 整体读起来是否自然流畅？
- [ ] 需要补充的真实文献/数据是否已明确标注？

## 注意事项

1. **【最高优先级】绝对禁止虚构**: 任何人名、文献、数据、案例都不得虚构，这是学术诚信的底线
2. **【硬性要求】不主动生成文档**: 除非用户明确要求，否则直接在对话中输出内容，不创建文件
3. **【推荐风格】口语化学术写作**: 在保持学术规范的前提下，使用更自然、更口语化的表达，让论文读起来更流畅
4. **保持学术规范**: 口语化不等于随意，要保持学术论文的基本规范和严谨性
5. **不过度口语化**: 避免使用过于随意的网络用语或俚语
6. **尊重原文意图**: 优化是为了更好地表达，而不是改变原意
7. **因文制宜**: 不同章节（摘要、引言、正文、结论）的写作风格可以有所差异
8. **真实性优先**: 宁可留空让用户补充，也不编造虚假内容

## 示例：研究现状优化

### 原文（过于正式）
```
在物流管理系统设计方面，相关学者进行了深入研究。某学者研究了数字化和智能化物流管理系统对物流效率的影响，研究结果表明信息化手段能够显著提升物流作业效率和管理水平。另有学者探讨了物流信息技术支撑下现代企业物流管理系统的构建策略，提出了基于信息技术的物流管理系统架构。此外，还有学者研究了智慧供应链导向的物流管理系统，强调了供应链协同在物流管理中的重要性。
```

### 优化后（口语化学术）
```
在物流管理系统设计方面，韦干研究了数字化和智能化物流管理系统对物流效率的影响，指出信息化手段能显著提升物流作业效率和管理水平[9]。岳谨鲜等人探讨了物流信息技术支撑下现代企业物流管理系统的构建策略，提出了基于信息技术的物流管理系统架构[10]。孙晓明等人研究了智慧供应链导向的物流管理系统，强调了供应链协同在物流管理中的重要性[11]。
```

**优化要点**：
1. 使用真实文献引用，不虚构
2. 用"这方面"、"指出"等更自然的表达
3. 避免"某学者"、"另有学者"的机械表述
4. 长短句结合，读起来更流畅
